## 概述

### 问题背景

全球约7.7亿低识字成年人（中国约4000万），加上10-20%的阅读障碍患者和视力衰退的老年人，面临严重的信息获取障碍：

- **药品安全风险**：看不懂药品说明书上的剂量、禁忌，导致误服
- **行政合同陷阱**：签租房合同、贷款协议时无法独立理解条款
- **出行公共服务障碍**：看不懂公交/地铁线路表、医院挂号流程
- **数字化排斥**：无法独立使用需要阅读的App和网站

> 一位50岁的低识字农村妇女，因为看不懂药品说明书上的"每日三次，每次2片"和"饭后服用"，误将药品空腹服用，导致严重胃出血。这类本可避免的事故每天都在发生。

### 解决方案

VoiceRead AI 是一款语音优先的AI助手，用户通过手机摄像头拍摄文字内容（药品标签、公交站牌、账单、合同），AI即时用简单口语朗读并解释含义。

**核心差异化**：不是翻译文字而是**解释含义**，用用户能理解的语言重新表达并主动标记重要信息。

### 目标用户

- **主要用户**：低识字成年人（全球7.7亿+）
- **次要用户**：阅读障碍患者（占比10-20%）、视力衰退的老年人、外务工者
- **痛点强度**：🔴 高频刚需 — 涉及安全、健康、法律等核心生活场景

## 技术方案

### 核心架构

```
┌─────────────┐    ┌──────────────┐    ┌───────────────┐    ┌──────────────┐
│  摄像头拍照  │ →  │  PaddleOCR   │ →  │  GLM-4 LLM    │ →  │  Edge-TTS    │
│  图片输入    │    │  文字识别     │    │  简化+解释     │    │  语音朗读     │
└─────────────┘    └──────────────┘    └───────────────┘    └──────────────┘
                                                ↕
                                        ┌───────────────┐
                                        │  Whisper STT   │
                                        │  语音输入       │
                                        └───────────────┘
```

### 关键AI能力

| 能力 | 技术实现 | 说明 |
|------|---------|------|
| 多模态OCR | PaddleOCR | 支持中文、英文、混合文字 |
| 内容简化 | GLM-4/GPT-4o | 用小学三年级语言重写 |
| 语音合成 | Edge-TTS | 支持方言（粤语、四川话、河南话） |
| 语音识别 | Whisper | 支持方言语音输入 |
| 场景识别 | GLM-4V | 自动识别文档类型 |
| 离线能力 | ONNX Runtime | 基础OCR+TTS支持离线 |

### 技术栈

| 层级 | 推荐技术 | 备选方案 |
|------|---------|---------|
| 前端 | 微信小程序 | Flutter / React Native |
| OCR | PaddleOCR | Tesseract / Google Vision |
| LLM | GLM-4 API | GPT-4o / Claude |
| TTS | Edge-TTS | Azure TTS / 讯飞 |
| STT | Whisper | 讯飞 / Azure Speech |
| 离线推理 | ONNX Runtime | TensorRT Lite |

## 市场与竞品分析

### 市场规模

- 全球功能性文盲人口 **10亿+**
- 美国4800万成年人阅读能力低于小学水平
- 中国低识字成年人约 **4000万**
- Legal Tech + 无障碍辅助技术市场快速增长

### 竞品分析

| 竞品 | 功能 | 局限 | VoiceRead差异 |
|------|------|------|-------------|
| Google Lens | OCR + 翻译 | 纯翻译，不理解含义 | 解释含义而非翻译 |
| 语音助手 | 语音交互 | 需精确提问 | 拍照即用，零门槛 |
| OCR App | 文字提取 | 只提取不理解 | 提取+简化+解释+朗读 |
| 讯飞听书 | TTS朗读 | 需手动输入文本 | 拍照自动识别 |

### 差异化壁垒

1. **理解上下文**而非朗读文字 — 核心技术壁垒
2. **方言+口语化**支持，贴近目标用户
3. **场景智能识别**，主动标记紧急/重要信息
4. **离线+云端**混合架构，适应弱网环境
