# AI 咖啡风味探索师

> **一句话卖点**：咖啡爱好者拍下豆袋标签，AI 引导品鉴训练，从只喝拿铁到盲喝猜产区

## 概述

基于 Issue #190 创建的详细设计文档。

AI 咖啡风味探索师是一款面向精品咖啡爱好者的品鉴能力培养工具。通过拍照识豆、风味训练、个性化推荐和盲喝挑战等功能，帮助用户从「喝不懂」到「能品鉴」，从「只喝拿铁」到「盲喝猜产区」。

产品不是卖豆子平台，而是培养品鉴能力的工具。通过 AI 引导用户建立风味词汇库，让用户能精准表达自己的喜好，最终成为真正的精品咖啡爱好者。

## 功能设计

### 核心功能

1. **拍照识豆**
   - 拍摄豆袋标签或咖啡馆菜单
   - AI 识别并解释产区、处理法、烘焙度
   - 展示预期风味特点（花香、果酸、甜感等）
   - 关联用户历史喜好，给出个性化解读

2. **风味训练**
   - 每次品鉴时 AI 引导用户用标准词汇描述
   - 提供「这是草莓甜，不是焦糖甜」等对比示例
   - 帮助用户建立风味词汇库
   - 记录用户的品鉴笔记和感受

3. **个性化推荐**
   - 基于历史品鉴数据，推荐「你可能喜欢的豆子」
   - 提供「推荐理由」建立信任（"上次你喜欢水洗肯尼亚的明亮酸质，这支的甜感会更饱满"）
   - 支持按产区、处理法、风味特点筛选
   - 关联购买渠道（合作电商或咖啡馆）

4. **盲喝挑战**
   - 随机推荐豆子进行盲喝测试
   - AI 引导对比品鉴（"现在喝的是 A，接下来喝 B，注意感受酸度的变化..."）
   - 记录品鉴结果和进步曲线
   - 游戏化设计，增加学习动力

5. **风味笔记**
   - 自动生成品鉴记录
   - 支持文字、图片、评分
   - 导出为 PDF 或分享到社交媒体
   - 下次买豆有依据

### 用户画像

- **主要用户**：想探索精品咖啡但不知道怎么选的爱好者
- **痛点**：喝不出区别、不会选、表达不出、进步慢
- **场景**：咖啡馆选豆、购买豆子、品鉴学习

## 技术方案

### 数据层

1. **咖啡知识图谱**
   - 产区数据库（埃塞俄比亚、肯尼亚、哥伦比亚等）
   - 处理法数据（水洗、日晒、蜜处理等）
   - 风味轮数据（SCA 标准 + 中文本地化）
   - 烘焙度与风味关联

2. **用户画像**
   - 品鉴历史记录
   - 风味偏好向量（酸/甜/苦/花香/果香等）
   - 进步曲线数据

3. **豆子数据库**
   - 豆子基本信息（产区、处理法、烘焙度）
   - 风味描述（专业标注 + 用户反馈）
   - 购买渠道链接

### 逻辑层

1. **OCR 识别模块**
   - 豆袋标签识别（GLM-OCR 或 Qianfan-OCR）
   - 菜单文字识别
   - 品牌和产品名提取

2. **风味分析模块**
   - 基于豆子信息的预期风味生成
   - 用户描述的语义分析
   - 风味词汇匹配和推荐

3. **个性化推荐引擎**
   - 冷启动：规则引擎（怕苦→推荐浅烘焙）
   - 迭代期：用户反馈 → 微调偏好模型
   - 成熟期：协同过滤（"和你口味相似的人喜欢..."）

4. **品鉴引导系统**
   - 对比品鉴流程设计
   - 风味词汇提示
   - 进步追踪算法

### 展示层

1. **移动端 App**（主要入口）
   - 拍照识豆界面
   - 品鉴引导流程
   - 个人中心（历史记录、偏好设置）

2. **Web 端**（数据查看和管理）
   - 品鉴记录查看
   - 数据导出
   - 社区分享

## 实现步骤

### Phase 1: MVP（2-3 周）

1. **核心功能**
   - 拍照识豆（OCR + 基础知识库）
   - 简单的风味引导
   - 品鉴记录保存

2. **数据准备**
   - 接入 Coffee Review / SCA 风味轮数据
   - 构建基础咖啡知识图谱（50-100 种常见豆子）
   - 众包标注启动

3. **技术栈**
   - 前端：React Native / Flutter
   - 后端：Node.js + Express
   - 数据库：PostgreSQL + Redis
   - AI：GLM-4 + OCR API

### Phase 2: 个性化推荐（3-4 周）

1. **用户画像构建**
   - 品鉴历史分析
   - 风味偏好建模

2. **推荐系统**
   - 规则引擎（冷启动）
   - 协同过滤（迭代期）

3. **购买渠道对接**
   - 合作电商平台 API
   - 咖啡馆数据接入

### Phase 3: 盲喝挑战与社区（4-5 周）

1. **盲喝系统**
   - 随机推荐算法
   - 对比品鉴流程
   - 进步追踪

2. **社区功能**
   - 品鉴笔记分享
   - 用户互动（点赞、评论）
   - 排行榜

### Phase 4: 商业化（5-6 周）

1. **付费功能**
   - Pro 版本（盲喝挑战 + 高级推荐）
   - Premium 版本（1v1 教练 + 豆包配送）

2. **B2B2C**
   - 咖啡馆合作
   - 电商嵌入

## 资源需求

- **AI 服务**
  - GLM-4 API（风味分析、用户引导）
  - OCR API（豆袋识别）

  - 数据库服务

## 变现模式

### B2C 订阅

```
免费版：每日3次识豆 + 基础风味训练
Pro ¥29/月：盲喝挑战 + 个性化推荐 + 风味笔记导出
Premium ¥99/月：1对1 AI 品鉴教练 + 季度豆包配送
```

### B2B2C 合作

1. **咖啡馆合作**
   - 作为店内「智能选豆助手」
   - 收费模式：SaaS 订阅或按使用量计费

2. **电商嵌入**
   - 嵌入购买流程，推荐适合的豆子
   - 收费模式：CPS（成交分成）

3. **企业福利**
   - 作为员工福利（每天一杯咖啡的费用）
   - 收费模式：企业订阅

### 数据价值

- 中国咖啡消费者的风味偏好数据
- 为咖啡豆进口商提供市场洞察
- 为咖啡品牌的产品研发提供数据支持

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **风味数据冷启动** | 初期接入 Coffee Review / SCA 数据；众包标注；与精品咖啡品牌合作获取专业数据 |
| **用户画像精准度** | 「只喝拿铁」需要区分细分原因（怕苦/怕酸/喜欢奶味）；通过引导式问答收集精准信息 |
| **推荐可信度** | 提供「推荐理由」解释；引入「盲喝测试」验证机制；展示相似用户的评价 |
| **OCR 准确性** | 建立「豆袋标签数据集」专门训练识别模型；支持手动修正；用户反馈优化 |
| **用户留存** | 游戏化设计（盲喝挑战、排行榜）；社区互动；持续的内容更新 |
| **竞争压力** | 聚焦「品鉴能力培养」差异化定位；先深耕「怕苦人群」单一痛点 |

## 成功指标

### MVP 阶段

- 日活跃用户 > 500
- 识豆功能使用率 > 60%
- 用户留存率（7日）> 30%

### 增长阶段

- 月活跃用户 > 10,000
- 付费转化率 > 5%
- 用户 NPS > 50

### 成熟阶段

- 月营收 > ¥100,000
- B2B2C 合作伙伴 > 10
- 品牌认知度（咖啡爱好者中）> 30%
