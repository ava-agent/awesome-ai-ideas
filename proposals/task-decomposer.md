# AI 任务拆解启动器

> **一句话卖点**：拖延症重度患者盯着任务焦虑刷手机时，用 AI 3 秒拆成 5 分钟微步骤，从「动不了」到「已经写了 2 小时」—— 只显示第一步，让开始变得毫无压力

## 概述

基于 Issue #106 创建的详细设计文档。

拖延不是懒，是「启动恐惧」—— 任务太大，大脑不知道从哪开始，于是选择逃避。本产品通过 AI 智能拆解任务，只显示第一步，降低启动门槛，帮助拖延症患者建立行动惯性。

## 功能设计

### 核心功能

- **⚡ 3 秒智能拆解** - 输入任务，AI 自动拆成 5-15 分钟微步骤
- **🎯 第一步聚焦** - 只显示当前步骤，降低心理压力
- **🔮 阻力预测** - 识别你可能卡住的地方，提前给提示
- **💪 个性化调整** - 学习你的拖延模式，定制拆解策略
- **🎉 完成反馈** - 每完成一步给正向激励，建立行动惯性

### 差异化功能

| 现有方案 | 本产品 |
|---------|--------|
| 番茄钟需要自己拆任务 | AI 3 秒自动拆好，不用动脑 |
| 看着一堆任务更焦虑 | 只显示第一步，降低启动门槛 |
| 倒计时增加紧迫感 | 正向激励，完成一步夸一步 |
| 通用时间管理工具 | 学习你的拖延模式，个性化拆解 |

### 核心算法

#### 1. 任务复杂度评估

```python
def assess_task_complexity(task: str, user_context: dict) -> ComplexityScore:
    """
    评估任务复杂度，决定拆解粒度
    
    维度：
    - 子任务数量
    - 依赖关系
    - 技术难度
    - 情绪阻力（基于用户历史）
    """
    llm_analysis = llm.analyze(task, aspects=[
        "estimated_time",
        "required_skills", 
        "emotional_barrier",
        "subtasks_count"
    ])
    
    user_factor = get_user_procrastination_factor(user_context)
    
    return ComplexityScore(
        granularity=calculate_granularity(llm_analysis, user_factor),
        predicted_barriers=identify_barriers(llm_analysis)
    )
```

#### 2. 个性化拆解策略

| 用户类型 | 拆解策略 | 步骤时长 | 激励风格 |
|---------|---------|---------|---------|
| 完美主义者 | 拆得更细，每步都有明确标准 | 5-8 分钟 | 强调「足够好」 |
| 冲动型 | 设置短时间限制，快速反馈 | 3-5 分钟 | 游戏化，连击奖励 |
| 焦虑型 | 强调「最小可行步骤」 | 8-12 分钟 | 温柔鼓励，降低压力 |
| 逃避型 | 先做最简单的部分 | 3-5 分钟 | 庆祝每个小胜利 |

#### 3. 阻力预测

```python
def predict_barriers(task: str, user_history: list) -> list[Barrier]:
    """
    预测用户可能在哪些步骤卡住
    
    基于因素：
    - 历史卡点模式
    - 任务类型相似度
    - 时间段（用户在某些时间更容易卡住）
    """
    similar_tasks = find_similar_completed_tasks(task, user_history)
    common_stuck_points = extract_stuck_patterns(similar_tasks)
    
    return [
        Barrier(
            step_index=idx,
            reason=predicted_reason,
            suggestion=generate_suggestion(predicted_reason)
        )
        for idx, predicted_reason in common_stuck_points
    ]
```
