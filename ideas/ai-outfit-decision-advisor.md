# AI 穿搭决策顾问

> **一句话卖点**：职场女性早上站在衣柜前，用 AI 穿搭顾问拍照说需求，从纠结试5套到3秒获得3套专业搭配建议

## 概述

基于 Issue #178 创建的详细设计文档。

每天早上，数以百万计的人在衣柜前纠结——试了一套又一套，最后还是穿了万年不变的那件。**AI 穿搭决策顾问** 让用户拍照上传自己的衣服，结合天气、场合、心情等场景因素，3秒内给出个性化搭配方案。

## 功能设计

### 核心功能

1. **衣柜数字化**
   - 拍照自动识别服装类型、颜色、风格
   - 智能分类：上衣/下装/外套/配饰/鞋履
   - 支持批量导入（从电商平台同步购买记录）

2. **场景感知推荐**
   - 天气联动：温度、湿度、降雨概率
   - 场合识别：工作/约会/运动/居家
   - 日程整合：读取日历识别重要会议
   - 心情输入：语音/文字快速描述

3. **个性化学习引擎**
   - 记录用户选择偏好
   - 学习风格标签（简约/时尚/运动/正式）
   - 反馈循环：喜欢/不喜欢/收藏

4. **搭配理由说明**
   - 解释为什么这样搭
   - 色彩搭配原理
   - 场合适配度评分
   - 培养用户穿搭能力

5. **新衣搭配建议**
   - 买新衣服时拍照
   - 立即显示衣柜中可搭配单品
   - 避免闲置浪费

## 技术方案

### 数据层

```
用户数据
├── 衣柜 (wardrobe)
│   ├── 单品ID、图片URL、类型、颜色、风格标签
│   ├── 购买时间、价格、品牌
│   └── 穿着次数、上次穿着日期
├── 偏好 (preferences)
│   ├── 风格倾向、颜色偏好
│   └── 品牌偏好、场合偏好
└── 历史 (history)
    ├── 每日穿搭记录
    ├── 用户反馈（喜欢/跳过）
    └── 外部评价（可选）
```

### 逻辑层

1. **图像识别服务**
   - 多模态模型：识别服装属性
   - 调用 GPT-4V / Claude Vision / 本地模型

2. **推荐引擎**
   - 规则引擎：场合 × 天气 × 风格 → 候选集
   - 排序模型：基于历史偏好重排序
   - 多样性保证：避免重复推荐

3. **上下文整合**
   - 天气 API（和风天气）
   - 日历读取（Google Calendar / Apple Calendar）
   - LLM 理解用户自然语言需求

### 展示层

- **移动端优先**：iOS/Android 原生或 React Native
- **拍照界面**：简洁的单品上传流程
- **推荐卡片**：3套方案并排展示，滑动切换
- **详情页**：搭配理由 + 穿着历史

## 资源需求

### API
- **多模态 LLM**：GPT-4V / Claude Vision（服装识别）
- **天气 API**：和风天气 / OpenWeather
- **日历 API**：Google Calendar API

### 服务
- **图片存储**：AWS S3 / Cloudflare R2
- **用户数据**：PostgreSQL / Supabase
- **推荐引擎**：自建或使用推荐服务

### 预估成本（月活 1 万）
- 多模态 API：$500-1000
- 存储和数据库：$100-200
- 总计：$600-1200/月

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 冷启动阻力大，用户懒得拍照 | 1. 接入淘宝/京东购买记录自动导入<br>2. 渐进式录入，首次只需 10 件 |
| 多模态 API 成本高 | 1. 先用图文诊断降本<br>2. 建立本地缓存减少重复调用 |
| 用户隐私顾虑 | 1. 本地优先处理<br>2. 明确数据使用范围，支持随时删除 |
| 竞品模仿 | 1. 个性化学习壁垒<br>2. 社区网络效应 |
| 推荐不准影响体验 | 1. 明确反馈机制<br>2. 冷启动期提供人工审核选项 |

## 成功指标

- **DAU/MAU**：日活用户占比 > 30%

- **付费转化**：免费 → 付费转化率 > 5%
- **留存率**：30 日留存 > 40%
