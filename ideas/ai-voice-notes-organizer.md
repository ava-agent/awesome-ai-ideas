# AI 语音笔记结构化助手 - 让零散想法自动井井有条

> 一句话描述：语音输入自动转为结构化笔记，智能分类、关联知识、保留情感线索

## 功能设计

### 核心功能
- **智能分类** - 自动识别内容类型（想法/待办/会议记录/灵感/备忘）
- **结构化输出** - 生成带标题、标签、优先级的格式化笔记
- **知识关联** - 检测与已有笔记的关联，建立链接
- **情感标记** - 保留语音中的情绪线索（兴奋/犹豫/紧急）
- **定时提醒** - 重要待办自动推送到日历/任务管理

### 用户场景
1. 通勤路上记录工作想法 → 自动整理成待办清单
2. 会议中快速语音笔记 → 结构化会议纪要
3. 睡前灵感捕捉 → 第二天早上收到整理好的笔记
4. 随时随地记录 → 统一的知识库

## 技术方案

### 数据层
- **语音存储** - 本地缓存原始音频，支持重新处理
- **笔记索引** - 使用向量数据库（如 Qdrant/Chroma）存储笔记嵌入
- **关联图谱** - 记录笔记间关联关系

### 逻辑层
- **ASR 服务** - 集成 Whisper/讯飞/阿里云语音识别
- **分类引擎** - LLM 驱动的内容分类 + 规则增强
- **关联检测** - 语义相似度计算 + 关键词匹配
- **情感分析** - 语音语调分析 + 文本情感识别

### 展示层
- **移动端** - 快速语音输入，实时预览结构化结果
- **Web 端** - 笔记管理、知识图谱可视化
- **集成** - Notion/Obsidian/飞书 插件

## 实现步骤

### Phase 1: MVP (2 周)
1. 语音转文字基础能力（Whisper API）
2. 简单分类：待办 vs 想法 vs 备忘
3. 基础结构化输出（标题 + 正文 + 标签）
4. 本地存储 + 简单搜索

### Phase 2: 核心功能 (3 周)
1. 5 类内容精准分类
2. 知识关联检测
3. 情感标记功能
4. Notion/Obsidian 插件

### Phase 3: 增强功能 (2 周)
1. 待办推送日历/任务管理
2. 笔记模板市场
3. 团队协作功能

## 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 用户语音输入习惯需培养 | 冷启动难 | 提供文字输入降级方案 |
| 分类准确率不足 | 体验差 | 人工反馈 + 持续优化 |
| 隐私担忧 | 用户流失 | 强调本地处理，数据不上云 |
| 笔记工具 API 依赖 | 集成受限 | 多平台并行对接 |
