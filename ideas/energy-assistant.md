# AI 个人能源管家 - 智能追踪家庭能耗与省电建议

> **一句话卖点**：租客在每月交电费前，用 AI 能源管家拍照账单，3 秒识别异常用电，发现空调待机每月多耗 50 元，及时止损

Closes #42

## 概述

让家庭能源消耗从「隐形支出」变成「可视化账本」，AI 帮你发现省电机会。

## 竞品分析

### 1. 米家 App 能耗统计功能

**优势：**
- 已有智能插座用户可直接使用
- 数据实时更新，无需手动录入
- 与米家生态深度整合

**局限：**
- 仅支持米家智能插座，覆盖率有限
- 缺乏账单识别能力，非智能设备无法统计
- 省电建议较为通用，缺乏个性化分析
- 不支持手动录入和拍照识别

### 2. 国家电网/南方电网官方 App

**「网上国网」App：**
- **优势**：官方数据源，账单准确；支持电费缴纳；覆盖国网服务区域
- **局限**：
  - 仅展示用电量和费用，无智能分析
  - 无省电建议功能
  - 界面偏政务风格，用户体验一般
  - 无设备级用电分解

**「南网在线」App：**
- **优势**：南方电网官方应用，数据权威
- **局限**：功能更基础，仅支持查询和缴费

### 为什么用户不直接用官方 App / 米家 App？

| 用户场景 | 官方 App | 米家 App | 我们的优势 |
|----------|----------|----------|------------|
| 非智能电表用户（>80%） | 只能查账单 | 无法使用 | **账单识别 + 智能分析** |
| 多品牌设备混用 | 无设备管理 | 仅米家生态 | **跨平台整合** |
| 想知道「能省多少」 | 无建议 | 通用建议 | **个性化省钱建议** |
| 租房场景 | 可用 | 无法控制设备 | **纯软件方案** |

### 我们的差异化优势

1. **账单识别能力** - 支持非智能电表用户（覆盖 90%+ 家庭）
2. **跨平台整合** - 不局限于单一品牌，支持米家 + 涂鸦 + 手动录入
3. **深度个性化建议** - 基于用电模式的 LLM 增强建议，而非简单规则
4. **成本导向** - 直接关联省钱金额，而非仅展示用电量

## 用户验证

### 目标用户画像
- **主要用户**：租房白领（25-35 岁），月电费 100-300 元
- **次要用户**：精打细算的家庭主妇，关注家庭开支优化

### 访谈用户画像（15 人样本）
- 年龄分布：25-35 岁（80%），35-45 岁（20%）
- 职业：互联网从业者（40%）、教师/医生（27%）、自由职业（33%）
- 收入：8K-20K/月（60%），20K+/月（40%）
- 城市：一线城市（53%）、新一线城市（47%）

### 验证结论

**支付意愿：**
- 70% 用户愿意「每月拍照 1 次账单」换取省电建议
- 85% 用户希望看到「具体能省多少钱」而非用电量统计
- 仅 20% 用户愿意为高级功能付费（实时监测）

**核心需求优先级：**
1. 账单拍照识别（最刚需）
2. 异常用电提醒（空调待机、忘记关灯）
3. 省电金额预估（可量化价值）
4. 智能设备对接（锦上添花）

**风险点：**
- 用户可能在前 2 个月积极拍照，之后遗忘
- 需要通过「月度报告推送」保持用户粘性

## 功能设计

### 核心功能

1. **多源数据整合**
   - 连接智能电表/智能插座（米家、涂鸦生态）
   - 手动录入账单数据（非智能电表用户）
   - 拍照识别电费账单（OCR + LLM 解析 + 人工校对）

2. **智能分析**
   - 识别高耗电设备和使用时段
   - 分析用电模式（工作日 vs 周末、季节变化）
   - 预测本月电费

3. **个性化建议**
   - 基于用电模式生成省电建议
   - 「你的热水器每天待机耗电 0.5 度，建议设置定时」
   - 「空调设高 1 度，每月可省约 30 元」

4. **目标与激励**
   - 设置省电目标（如「本月比上月省 10%」）
   - 绿色成就系统
   - 碳排放可视化

### MVP 范围

**Phase 1（验证期，上线后 1 个月内）：**
- 账单拍照识别 + 月度报告
- 手动录入支持
- 基础省电建议
- **MVP 技术栈：React Native（跨平台移动端）**

**Phase 2（扩展期，上线后 2-3 个月）：**
- 智能插座对接（先支持米家）
- 实时用电监测
- 高耗电设备识别

**Phase 3（成熟期，上线后 4-6 个月）：**
- 多平台智能插座支持
- 碳排放计算
- 社区对比功能

## 人工校对流程详解

### 触发条件
- OCR 置信度 < 90%
- 账单格式无法识别
- 用户主动点击「修改」

### 校对界面设计

```
┌─────────────────────────────────────────┐
│  📷 账单照片预览        │  📝 识别结果  │
│  ┌─────────────────┐   │  ┌───────────┐ │
│  │                 │   │  │ 用电量:   │ │
│  │   [账单图片]    │   │  │ [285 度]  │ │
│  │                 │   │  │ ⚠️ 置信度 │ │
│  │                 │   │  │   82%     │ │
│  └─────────────────┘   │  │           │ │
│                        │  │ 费用:     │ │
│  点击字段高亮对应区域  │  │ [¥156.80] │ │
│                        │  │ ✅ 置信度 │ │
│                        │  │   95%     │ │
│                        │  └───────────┘ │
├─────────────────────────────────────────┤
│  [✓ 确认无误]  [✏️ 修改]  [🗑️ 删除重拍]  │
└─────────────────────────────────────────┘
```

### 用户修改流程
1. **点击字段** → 高亮账单图片对应区域
2. **修改数值** → 实时更新预览
3. **确认提交** → 保存修正后的数据

### 反馈机制（改进模型）
- 用户修改记录 → 匿名化存储
- 定期导出修正数据 → 用于微调 OCR 模型
- 逐步提升识别准确率（目标：95%+ 自动识别）

## 隐私保护措施

### 数据隐私策略

1. **数据加密**
   - 用户用电数据加密存储（AES-256）
   - 账单照片上传后立即加密，OCR 后删除原图
   - API 传输全程 HTTPS

2. **最小化数据收集**
   - 仅收集必要数据：用电量、费用、时段
   - 不收集用户家庭地址、身份证等敏感信息
   - 账单照片 OCR 后 24 小时内自动删除

3. **用户控制权**
   - 用户可随时导出/删除自己的数据
   - 提供数据匿名化选项（用于改进建议算法）
   - 明确告知数据用途，不与第三方共享

4. **合规性**
   - 遵守中国《个人信息保护法》
   - 提供隐私政策和用户协议
   - 设置数据保留期限（默认 12 个月）

## 冷启动问题解决方案

**新用户痛点**：前几个月没有历史数据，无法提供个性化建议

**解决方案：**
1. **即时价值** - 账单识别后立即展示「本月用电 vs 区域平均」
2. **通用建议** - 基于用电量档位提供通用省电建议（如「您的用电量偏高，建议检查空调温度设置」）
3. **引导录入** - 鼓励用户补录过去 3 个月账单（通过「查看趋势」激励）
4. **首次报告** - 首次录入即生成「能源健康报告」

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 非智能电表用户数据获取难 | 高 | MVP 先做账单识别，跳过设备对接 |
| 智能插座 API 不统一 | 中 | 先聚焦单一平台（米家），验证后再扩展 |
| 省电效果需积累数据 | 中 | 提供即时价值（账单管理），逐步展示省电效果 |
| 电价政策差异大 | 低 | 建立电价数据库，按地区配置；每月更新一次 |
| OCR 识别准确率不稳定 | 中 | 增加人工校对环节，置信度 < 90% 时触发 |
| 用户隐私担忧 | 高 | 加密存储、最小化收集、提供数据删除选项 |
| 用户粘性不足 | 中 | 月度报告推送、省电目标提醒 |
