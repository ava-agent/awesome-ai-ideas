# AI 维权证据助手

> **一句话卖点**：消费者拍照上传商品问题，AI 3 秒生成专业投诉文书 + 维权路径指引，从投诉无门到证据齐全一击必中

## 概述

基于 Issue #107 创建的详细设计文档。

消费者维权痛点明显：遇到商品质量问题、服务欺诈、退款纠纷时，不知道从何下手。证据整理麻烦、投诉渠道不清、话术不专业，很多人为了几百块选择自认倒霉。

AI 维权证据助手通过 **拍照上传 → AI 整理证据链 → 一键生成投诉文书 → 维权路径指引** 的完整流程，让普通人也能高效维权。

## 功能设计

### 核心功能

1. **证据拍照上传，自动归档**
   - 拍商品问题、聊天记录、订单截图
   - AI 自动识别证据类型并分类
   - 生成证据链 ID，关联同一案件的所有材料
   - 每张照片自动记录时间戳，支持导出带时间戳和哈希值的证据包

2. **智能问题识别**
   - 多模态识别：商品照片 + OCR 商品标签 + 聊天记录
   - 自动分类：质量问题 / 虚假宣传 / 服务欺诈 / 价格欺诈
   - 给出参考案例相似度和维权成功率参考（30 秒快速评估，基于历史案例，不构成法律承诺）

3. **一键生成投诉文书**
   - 根据问题类型，生成专业的投诉信 / 举报材料
   - 要点清晰、有法可依，引用相关法规条款
   - 支持多种格式：PDF / Word / 纯文本

4. **维权路径指引**
   - 智能判断：应该先找商家 / 平台客服 / 12315 / 消协
   - 提供各渠道的投诉入口和注意事项
   - 一键复制话术模板，与商家 / 平台沟通

5. **进度追踪提醒**
   - 时间线管理：记录维权全过程
   - 节点提醒：不要错过关键时限
   - 快递式进度展示，减少用户焦虑

### 用户体验亮点

- **30 秒快速评估**：上传证据 → 立即给出「参考案例相似度」和「维权成功率参考」（基于历史案例，仅供参考）
- **进度可视化**：像快递追踪一样展示维权进度
- **一键复制话术**：和商家 / 平台沟通的模板，直接用

## 技术方案

### 数据层

- **证据存储**：PostgreSQL + 敏感信息加密存储
- **隐私分层处理**：
  - **本地处理**：OCR 识别、初步分类、证据归档（在用户设备完成，不上传原图）
  - **云端处理**：文书生成、法规匹配（仅上传脱敏后的文本摘要，不包含个人身份信息）
  - **用户可控制**：提供「纯本地模式」选项，仅用本地模型，功能受限但隐私最大化
- **证据链管理**：时间戳 + 哈希值确保证据完整性和时序性
- **案件库**：积累维权案例数据（脱敏），建立「消费者维权指数」

### 逻辑层

- **多模态识别**：GPT-4V 做初步分类，专门模型做细节提取
- **问题分类**：LLM few-shot learning，定义常见维权场景分类
- **文书生成**：Claude / GPT-4，结合法规数据库
- **路径决策**：规则引擎 + LLM 推荐

### 展示层

- **前端方案（按优先级）**：
  1. **H5 + PWA**（推荐）：无审核限制，支持添加到桌面，跨平台兼容
  2. **微信小程序**：需调研法律咨询类目资质要求，可能需要《增值电信业务经营许可证》
  3. **App**：作为长期方案，建立品牌护城河
- **关键交互**：拍照上传 → 3 秒反馈 → 一键生成 → 进度追踪
- **微信小程序风险预案**：如果小程序类目审核不通过，转为 H5 落地页 + 企业微信社群运营

## 实现步骤

### Phase 1：MVP（4 周）

- **Week 1-2**：核心功能开发
  - 证据拍照上传 + 自动归档
  - 智能问题识别（仅支持电商购物维权）
  - 一键生成投诉文书（3 大平台：淘宝 / 京东 / 拼多多）

- **Week 3-4**：用户体验优化 + 测试
  - 30 秒快速评估功能
  - 维权路径指引
  - 内测 + 修复 bug

### Phase 2：扩展场景（4 周）

- 增加外卖 / 餐饮维权场景
- 增加服务类维权（美容 / 健身 / 教育培训）
- 进度追踪 + 提醒功能

### Phase 3：商业化（持续）

- 对接律师服务（高价值案件引流）
- B2B 合作：与电商平台合作，作为商家合规检测工具
- 数据价值：发布「消费者维权指数」行业报告

## 资源需求

### API

- **多模态识别**：GPT-4V 或 Claude Vision
- **文书生成**：GPT-4 / Claude
- **OCR**：百度 OCR / 腾讯 OCR（识别商品标签、聊天记录）

### 服务

- **云服务**：阿里云 / 腾讯云（对象存储 + 数据库）
- **法规数据库**：
  - **首选**：对接北大法宝 API（权威法规数据库，覆盖消费者权益保护法、电子商务法等）
  - **备选**：自建法规库（爬取中国裁判文书网公开案例 + 法律条文，定期更新）
  - **MVP 阶段**：使用 LLM 内置法律知识 + 手动维护核心法规条文（50+ 常用条款）

### 技术栈

- **前端**：微信小程序 + Vue.js Web 管理后台
- **后端**：Node.js + PostgreSQL
- **AI**：GPT-4V + Claude

## 变现模式

### B2C 模式

- **免费基础功能**：证据拍照归档、问题分类、基础投诉模板（吸引流量）
- **增值服务**：
  - 专业文书生成：¥9.9 / 次（仅复杂案件需要）
  - 订阅制：¥19.9 / 月（无限次文书生成 + 进度追踪 + 优先客服）
- **成功后付费**：维权成功后收取 5% 服务费（小额纠纷 ¥50 以下免费）

### B2B 模式

- **电商平台合作**：维权成功后抽佣 5-10%
- **律师引流**：高价值案件（大额消费纠纷、人身伤害）导流给合作律师，收取佣金

### 数据价值

- 积累维权案例数据，发布「消费者维权指数」行业报告
- 建立影响力，吸引 B2B 合作

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 法律风险：AI 生成的投诉文书不能替代专业法律意见 | 明确免责声明，建议复杂案件咨询律师 |
| 隐私风险：用户上传的证据可能包含敏感信息 | 隐私分层处理：OCR 和分类本地完成，云端仅处理脱敏文本；提供「纯本地模式」选项 |
| 商家对抗：如果产品影响力大，可能被某些平台 / 商家抵制 | 做好合规，与平台建立合作而非对抗关系 |
| 证据效力：AI 生成的证据包法律效力不确定 | 提供时间戳 + 哈希值，建议重要证据公证 |
| 小程序审核风险：法律咨询类目可能需要特殊资质 | 优先使用 H5 + PWA 方案，小程序作为补充渠道 |

## 成功案例库（规划）

设计一个「维权成功案例库」（脱敏后），让用户看到「别人是怎么维权的」，既能教育用户，又能建立信任。
