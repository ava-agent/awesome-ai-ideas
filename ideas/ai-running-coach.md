# AI 跑步陪练 - 从跑500米就喘到轻松完成首个5公里

> **一句话卖点**：跑步新手在每次晨跑时，用 AI 陪练获得实时语音鼓励和配速指导，从跑500米就喘到3周后轻松完成首个5公里

## 概述

基于 Issue #174 创建的详细设计文档

这是一个面向跑步新手的 AI 陪练产品，通过实时语音陪伴、动态配速指导和情绪激励，帮助零基础用户跨越「从0到5公里」的心理和生理障碍。

**核心价值**：不是冰冷的配速数据，而是有温度的「我陪着你跑」

## 功能设计

### 核心功能 1：实时语音陪跑
- 耳机实时鼓励：「还有200米，保持住！」、「调整呼吸，吸气-吸气-呼」
- 延迟 < 1秒，确保不影响跑步节奏
- 支持多种语音风格（温和型/激励型/教练型）

### 核心功能 2：动态配速指导
- 根据实时心率自动调整配速建议
- 基于心率区间训练（最大心率的 60-70% 为燃脂区）
- 动态提醒：「慢一点，保持这个节奏」、「可以稍微加速了」

### 核心功能 3：跑姿监测（Phase 2）
- 通过手机摄像头进行姿态估计
- 检测步频、步幅、身体前倾角度
- 实时提醒：「肩膀放松，核心收紧」

### 核心功能 4：情绪加油站
- 跑不动时的正向反馈：「你已经跑完2公里了！比上周进步30%」
- 疲劳检测 + 个性化鼓励语
- 社交压力转化：「朋友圈打卡模板已生成」

## 技术方案

### 数据层
- **穿戴设备 API**：Apple HealthKit / Google Fit / 华为 / 小米 / 佳明 API 获取心率数据
- **运动数据存储**：SQLite/本地存储（心率、配速、轨迹）
- **训练记录**：云端存储（训练进度、历史数据）
- **用户画像**：本地向量化存储（用户偏好、体能水平）

### 逻辑层
- **语音合成引擎**：Edge TTS/云端 TTS（延迟 < 1s）
- **配速算法**：心率区间计算 + LSTM 动态调整
- **对话管理**：LLM 生成个性化鼓励语 + 疲劳检测（基于心率变化和配速的综合判断）

### 展示层
- **移动端 App**（React Native/Flutter/iOS）
- **数据可视化**：心率曲线、配速趋势图
- **完成度徽章**
- **社区功能**（虚拟跑团、挑战赛）

## 实现步骤

1. MVP 阶段：支持 Apple Watch + 手机扬声器，验证核心体验
2. 逐步扩展：支持华为、小米、佳明等其他设备
3. Phase 2：上线跑姿监测功能
4. Phase 3：完善社区和社交功能

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 数据隐私敏感 | 本地优先架构 + 端到端加密 + 用户完全控制数据 |
| 户外网络不稳定 | 端侧设备 API，接入 Whisper 本机转写（离线语音识别） |
| 端侧技术难点 | 模型量化（4-bit），使用轻量语音包（预生成常用语音） |
| 实时检测准确率低 | 初期用手机摄像头做跑后分析，实时检测需额外穿戴设备 |
| 法律合规风险 | 明确「辅助工具」定位，添加免责声明，训练前咨询专业教练 |

## 技术挑战
