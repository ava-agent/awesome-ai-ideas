# AI 聚会召集官 - 智能社交活动协调平台

> **一句话卖点**：周末想聚会的组织者在微信群发一句话，AI 5分钟内锁定时间地点，从讨论3天定不下来到所有参与者确认完毕

## 概述

基于 Issue #201 创建的详细设计文档。

**核心问题**：朋友聚会本该简单快乐，但现实中时间协调、地点选择、活动决策让组织者筋疲力尽，讨论3天最后取消是常态。

**解决方案**：AI 聚会召集官 — 一句话召唤，AI 自动协调时间、地点、活动，5分钟完成从提议到确认。

## 功能设计

### 核心功能

1. **智能时间匹配**
   - 连接参与者日历（可选）或群内回复
   - 自动找出所有人都有空的时间窗口
   - 考虑通勤时间、作息偏好

2. **地点偏好分析**
   - 根据历史记录 + 简单问答，推荐大家都满意的地方
   - 考虑地理位置（中位点）、交通方式、人均预算
   - 支持餐厅/咖啡厅/活动场所等分类

3. **活动类型匹配**
   - 识别聚会目的（聊天/庆祝/团建/约会）
   - 匹配合适的场景和氛围
   - 推荐具体商家/场地

4. **自动跟进**
   - 发出邀约后追踪回复
   - 催促未确认的人
   - 统计确认人数

5. **备选方案**
   - 主方案行不通时自动生成 Plan B
   - 多个方案并行投票

### MVP 功能（第一阶段）

- 群聊机器人接入（微信群/Discord）
- 时间协调（收集回复 → 找最佳时间）
- 地点推荐（基于位置中位点 + 偏好标签）

## 技术方案

### 数据层

```
用户数据
├── 基础信息（位置、作息偏好）
├── 日历集成（Google/Apple Calendar API）
├── 历史聚会记录
└── 偏好标签（预算、菜系、活动类型）

聚会数据
├── 参与者列表
├── 时间选项 → 回复
├── 地点候选 → 投票
└── 最终方案
```

### 逻辑层

```
API Gateway
    ├── 消息解析服务（NLU）
    │   └── 提取：时间偏好、地点偏好、活动类型
    ├── 时间匹配引擎
    │   └── 算法：寻找最大交集时间窗口
    ├── 地点推荐引擎
    │   ├── 地理中位点计算
    │   ├── 商家匹配（大众点评 API）
    │   └── 偏好加权评分
    └── 通知服务
        ├── 群消息推送
        └── 私聊提醒
```

### 展示层

- **群聊机器人**：微信/Discord/Slack Bot
- **Web Dashboard**：组织者查看详情、手动调整
- **Mini App**：轻量级投票界面

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户隐私顾虑（日历权限） | 高 | 默认不要求日历权限，用「群内回复收集」替代 |
| 冷启动（无用户数据时推荐差） | 中 | 初始用规则推荐 + 用户偏好标签快速建立画像 |
| 群聊平台限制（微信 API 政策） | 高 | 多平台策略（Discord/Slack 并行），不依赖单一平台 |
| 商家数据准确性 | 中 | 多数据源交叉验证（大众点评 + 美团 + 用户反馈） |
| 用户习惯改变难度 | 中 | 设计极简交互（一句话触发），降低使用门槛 |
