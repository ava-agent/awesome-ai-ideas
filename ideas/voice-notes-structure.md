# AI 语音笔记结构化助手

> 让零散的语音想法自动井井有条

## 竞品分析

> ⚠️ Review 反馈补充 - 差异化定位

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|--------------|
| **Otter.ai** | 会议记录 | 实时转写、说话人识别 | 偏会议场景、分类弱 | 个人笔记场景、智能分类 |
| **Notion AI** | 文档助手 | 深度集成 Notion | 需手动触发、无语音 | 语音优先、自动结构化 |
| **飞书妙记** | 企业协作 | 会议记录、企业集成 | ToB 导向、个人场景弱 | ToC 轻量、个人知识管理 |
| **讯飞听见** | 语音转写 | 中文精准、速度 | 无结构化、纯转写 | 结构化 + 知识关联 |

**核心差异化**：
1. **语音 → 结构化知识** - 不是转写工具，是知识整理助手
2. **个人知识图谱** - 自动关联已有笔记，构建知识网络
3. **轻量即开即用** - 不依赖会议场景，随时记录灵感

## 隐私与数据安全

> ⚠️ Review 反馈补充 - 语音数据高度敏感

### 数据加密

- **传输加密**: TLS 1.3 全链路加密
- **存储加密**: AES-256 静态数据加密
- **密钥管理**: 环境变量 + 云密钥服务 (AWS KMS / Azure Key Vault)

### 语音数据处理

- **即时处理**: 语音上传后立即转写，不保留原始文件
- **自动清理**: 转写完成 1 小时内删除语音原始文件
- **用户控制**: 用户可随时手动删除所有数据

### 用户数据权利

- **数据导出**: 一键导出所有笔记 (JSON / Markdown)
- **数据删除**: 账户注销时完全删除所有关联数据
- **访问日志**: 用户可查看数据访问记录

### 数据迁移方案

> ⚠️ Review 反馈补充 - 本地到云端迁移

**问题**：MVP 第 1-3 周使用本地存储，第 4 周引入用户认证后需要迁移数据。

**方案**：
1. **本地存储结构**：使用 IndexedDB / SQLite，结构设计与云端一致
2. **迁移时机**：用户首次登录时触发迁移向导
3. **迁移流程**：
   - 检测本地数据 → 显示迁移确认 → 上传云端 → 验证成功 → 本地数据标记已迁移
4. **冲突处理**：时间戳优先，云端数据为准
5. **回退方案**：迁移失败时保留本地数据，用户可手动重试

```yaml
迁移实现:
  触发: 用户首次登录
  检测: IndexedDB 中是否有未迁移数据
  确认: UI 弹窗显示数据量，用户确认后执行
  上传: 批量上传到云端，显示进度条
  验证: 逐条校验云端数据完整性
  清理: 本地数据标记已迁移，不删除（保留备份）
  重试: 失败时记录断点，支持断点续传
```

## 功能设计

### 核心功能

1. **智能分类引擎**
   - 自动识别内容类型：想法/待办/会议记录/灵感/备忘
   - 多标签支持：一条笔记可同时属于多个类别
   - 置信度评分：当分类不确定时标注，供用户确认

2. **结构化输出**
   - 自动生成标题：从散乱语音中提取核心主题
   - 智能标签：基于内容关键词和上下文
   - 优先级推断：从语气词和时间敏感词判断紧急程度
   - 格式化输出：Markdown / Notion / Obsidian 格式

3. **知识关联网络**
   - 检测与已有笔记的语义关联
   - 自动建立双向链接
   - 标记重复或相关主题
   - 构建个人知识图谱

4. **情感线索保留**
   - 识别语音中的情绪标记（兴奋/犹豫/紧急/困惑）
   - 生成情感标签用于回顾
   - 情绪趋势分析

5. **智能提醒系统**
   - 自动识别待办事项并推送
   - 集成日历 / 任务管理工具
   - 基于紧急程度和截止日期的智能提醒

## 技术方案

### 数据层

```yaml
语音输入:
  - 格式: WebM / M4A / WAV
  - 存储: 云端临时存储，处理后删除

笔记数据:
  - 主库: PostgreSQL / Supabase
  - 向量库: Pinecone / Qdrant (语义搜索)
  - 缓存: Redis (热点数据)

用户配置:
  - 偏好设置: 笔记格式、默认分类
  - 集成配置: Notion/Obsidian/日历 API keys
```

### 逻辑层

```yaml
语音处理:
  ASR: 
    - 首选: Whisper Large V3 (高精度)
    - 备选: 讯飞语音 (中文优化)
  
  情感分析:
    - Whisper 的时间戳 + 音高/语速特征
    - 情感分类模型 (兴奋/犹豫/紧急/平静)

文本处理:
  分类器:
    - LLM: GPT-4 / Claude 3.5
    - Prompt: 结构化分类 + 置信度输出
  
  结构化:
    - 标题生成: LLM 提取核心主题
    - 标签提取: NER + 关键词 + LLM
    - 优先级: 规则 + LLM 判断
  
  关联检测:
    - 向量嵌入: text-embedding-3-small
    - 相似度阈值: 0.75
    - 图数据库: Neo4j (可选，用于复杂关联)

集成层:
  Notion API: 笔记同步
  Obsidian: Local REST API 插件
  日历: Google Calendar / Apple Calendar

错误处理:
  重试策略: 指数退避，最多 3 次
  降级方案: Whisper 失败时降级到备用 ASR
  错误日志: 结构化日志 + Sentry 告警
  用户通知: 处理失败时明确告知用户

并发处理:
  队列机制: Redis 队列管理并发请求
  限流策略: 用户级别速率限制 (10 次/分钟)
  优先级队列: 付费用户优先处理
  状态同步: WebSocket 实时推送处理进度
```

### 向量库选型

> ⚠️ Review 反馈补充

| 特性 | Pinecone | Qdrant |
|------|----------|--------|
| 部署 | 托管服务 | 自托管 / 云托管 |
| 成本 | $70/月起 | 开源免费 / 按资源付费 |
| 扩展性 | 自动扩展 | 手动扩展 |
| 延迟 | 低 | 极低 (自托管) |
| 推荐场景 | 快速启动，无运维 | 成本敏感，可控部署 |

**MVP 推荐**: Pinecone (快速启动)
**长期推荐**: Qdrant 自托管 (成本优化)

### 展示层

```yaml
Web 端:
  - React / Next.js
  - 实时转写显示
  - 笔记编辑器 (支持 Markdown)

移动端:
  - PWA / React Native
  - 语音录制组件
  - 离线队列 (无网络时暂存)

离线队列技术方案:
  存储: IndexedDB 本地存储
  结构:
    - id: UUID
    - audio_blob: 语音二进制
    - created_at: 时间戳
    - status: pending/uploading/uploaded/failed
    - retry_count: 重试次数

  时序处理:
    - 按 created_at 排序，先进先出
    - 上传时携带原始时间戳，保留用户记录时间

  去重策略:
    - 上传成功前保留本地副本
    - 服务端返回唯一 ID 后标记本地为已同步
    - MD5 校验防止重复上传

  冲突解决:
    - 同一用户多设备：最后修改时间优先
    - 网络恢复后批量同步，显示冲突提示
    - 用户可选择保留哪个版本

  错误处理:
    - 网络失败：指数退避重试，最多 5 次
    - 服务端错误：记录错误日志，用户手动重试
    - 存储空间不足：提示用户清理旧录音

桌面端:
  - Electron (可选)
  - 系统托盘快速录音
```

## 实现步骤

### Phase 0: 超精简验证版 (1-2 周)

> ⚠️ Review 反馈新增 - 先验证核心假设

**目标**：用最小功能验证「语音 → 结构化笔记」是否有价值

**范围**：
1. **语音转写**
   - Web 录音界面（无 App）
   - Whisper API 集成
   - 转写结果展示

2. **基础分类**
   - 仅 3 类：想法 / 待办 / 备忘
   - LLM 分类（无置信度）

3. **本地存储**
   - IndexedDB 存储（无后端）
   - 简单列表展示
   - 无用户系统

**验证指标**：
- 自己使用 1 周，记录使用频率
- 核心问题：分类准确吗？真的节省时间吗？

**决策点**：如果验证通过，进入 Phase 1；否则调整方向或放弃

### Phase 1: MVP (3-4 周)

> ⚠️ Review 反馈调整 - 基于验证版扩展

**P0 - 核心功能 (第 1-2 周)**
1. **语音转写**
   - 集成 Whisper API
   - 简单的 Web 录音界面
   - 转写结果实时显示

2. **基础分类**
   - 3 类核心分类：想法/待办/备忘（精简）
   - LLM 分类 Prompt 设计
   - 简单的标签提取

3. **笔记存储**
   - 笔记 CRUD (无用户认证，使用简单 token)
   - 本地存储优先（降低初期复杂度）

**P1 - 结构化增强 (第 3 周)**
4. **结构化输出**
   - 自动生成标题
   - 优先级推断
   - Markdown 格式输出

5. **基础搜索**
   - 关键词搜索
   - 分类筛选

**P2 - 用户系统 (第 4 周)**
6. **用户认证**
   - 简单邮箱注册
   - 云端数据同步
   - 数据迁移支持（见隐私与数据安全章节）

**暂缓功能（移至后续阶段）**
- ❌ 情感分析 → Phase 3
- ❌ 知识关联 → Phase 2
- ❌ 智能提醒 → Phase 2

### Phase 2: 智能化 (3-4 周)

4. **知识关联**
   - 向量嵌入集成
   - 语义相似度计算
   - 相关笔记推荐

5. **智能提醒**
   - 待办识别优化
   - 日历集成
   - 提醒规则引擎

6. **外部工具集成**
   - Notion API 同步
   - Obsidian 插件
   - 飞书/钉钉集成

### Phase 3: 高级功能 (2-3 周)

7. **情感分析**
   - 语音情感特征提取
   - 情感标签生成
   - 情绪趋势可视化

8. **移动端优化**
   - PWA 离线支持
   - 后台录音
   - 推送通知

### Phase 4: 增强 (持续)

9. **高级功能**
   - 风格学习 (学习用户表达习惯)
   - 多语言支持
   - 团队协作

## 成本控制策略

### API 成本优化

| 策略 | 说明 | 预期节省 |
|------|------|----------|
| 模式缓存 | 常见分类模式缓存，减少 LLM 调用 | 30% |

- **免费**: 100 条/月 (原 50 条)，基础分类
- **Pro ($9.9/月)**: 1000 条/月，高级功能
