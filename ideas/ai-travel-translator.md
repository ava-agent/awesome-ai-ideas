# AI 境外游翻译官

> **一句话卖点**：只会中文的游客在泰国夜市点菜，用 AI 翻译官实时对话，从手舞足蹈到开口就是「当地人」

## 概述

基于 Issue #127 创建的详细设计文档。

解决境外游小白的核心痛点：语言不通、翻译软件难用、不懂当地文化。通过实时语音翻译 + 场景模式 + 文化提示，让只会中文的游客也能在境外自如沟通。

## 功能设计

### 核心功能

1. **实时语音对话翻译**
   - 说话自动识别语言，即时翻译并播放
   - 支持 50+ 语言（泰语、日语、韩语、法语、西班牙语等）
   - 对话模式：你一句我一句，AI 自动识别谁在说话

2. **场景模式优化**
   - **点菜模式**：拍菜单 → 识别菜品 → 翻译 + 推荐评分
   - **问路模式**：拍路牌 → 翻译 + 显示地图位置
   - **购物模式**：识别商品 → 翻译说明 + 砍价话术
   - **就医模式**：症状描述模板 + 紧急翻译

3. **拍照识别 + 智能翻译**
   - 菜单：不只是翻译，还解释「这道菜是什么、辣不辣、评分多少」
   - 路牌：翻译 + 显示周边地图
   - 说明书：翻译使用方法 + 注意事项

4. **文化提示**
   - 检测敏感话题，提醒当地禁忌
   - 例：「泰国寺庙不能穿短裤」「日本餐厅不能自己倒酒」

5. **离线短语包**
   - 核心短语离线可用
   - 常用场景（点菜、问路、砍价）预下载
   - 没网也能应急沟通

### 差异化功能

| 现有方案 | 本产品 |
|---------|--------|
| 通用翻译，需手动切换语言 | 说话自动识别场景，一步到位 |
| 只翻译字面意思 | 翻译 + 解释：「冬阴功」→「酸辣虾汤，有点辣」 |
| 拍照只出翻译结果 | 拍照 + 识别 + 翻译 + 推荐 |
| 必须联网才能用 | 核心短语离线可用 |

## 技术方案

### 数据层

- **语音识别**：Whisper API（OpenAI）或 讯飞语音
- **多语言翻译**：GPT-4 / Claude API，支持上下文理解
- **图像识别**：GPT-4V / GLM-4V，识别菜品、路牌、说明书
- **场景知识库**：旅游场景专用语料库（点菜话术、问路表达）

### 逻辑层

- **场景识别引擎**：基于对话内容判断当前场景（点菜/问路/购物）
- **翻译优化器**：不只是字面翻译，还添加场景化解释
- **文化检测模块**：检测敏感话题，提醒当地禁忌
- **离线缓存管理**：智能预下载常用短语包

### 展示层

- **语音交互界面**：按住说话，松开翻译
- **AR 相机模式**：相机对准菜单，实时叠加翻译
- **场景切换按钮**：快速切换点菜/问路/购物模式
- **离线状态指示**：明确显示哪些功能离线可用

### 数据

- **旅游场景语料库**：点菜话术、问路表达、砍价技巧
- **文化禁忌数据库**：各国禁忌、注意事项
- **菜品数据库**：热门旅游地的菜品信息

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Google Translate 免费且强大 | 聚焦「旅游场景深度优化」，做细分市场 |
| 实时翻译延迟 >3 秒 | 边缘计算 + 预加载常用翻译 |
| 离线模式占用存储空间 | 智能预下载，只下载用户即将前往的国家 |
| 各国口音识别困难 | 针对热门国家（泰、日、韩）做专项优化 |
| 语音合成不够自然 | 使用 ElevenLabs 或 Azure TTS 提升质量 |

## 竞品分析

| 竞品 | 优势 | 劣势 |
|------|------|------|
| Google Translate | 免费、语言多 | 无场景优化、无文化提示 |
| DeepL | 翻译质量高 | 语言少、无语音功能 |
| 百度翻译 | 中文优化好 | 无场景模式、无离线包 |
| iTranslate | 语音翻译好 | 收费贵、无文化提示 |

## 目标市场

### 一级市场

- **中国出境游小白**：年超 1 亿人次
- **只会中文、不敢开口**：约 60% 的出境游客
- **热门目的地**：泰国、日本、韩国、东南亚

### 二级市场

- **商务出差**：需要专业场景翻译
- **留学生家长**：探亲旅游需要翻译
- **银发族**：子女为父母购买
