# AI 搬家打包指挥官

> **一句话卖点**：租房年轻人在搬家前一周，用 AI 搬家指挥官拍照识别物品，从手忙脚乱打包3天到半天井然有序

## 概述

基于 Issue #165 创建的详细设计文档。

搬家是最让人头疼的琐事之一，现有方案（Excel/备忘录/搬家App）都需要手动输入，太费时间。本产品通过 AI 多模态识别，让用户只需拍照即可自动生成打包清单、智能排序、易碎品提醒，大幅降低搬家焦虑和时间成本。

## 功能设计

### 核心功能

1. **拍照识别物品**
   - 拍摄房间照片，AI 自动识别并分类 50+ 物品
   - 支持多张照片拼接，覆盖整个房间
   - 自动识别物品类型：书籍、衣物、电子产品、易碎品等

2. **智能打包顺序**
   - 根据房间、物品类型、使用频率生成最优打包顺序
   - 逻辑：不常用物品（书籍、换季衣物）→ 日常用品 → 易碎品
   - 目标：空间利用率最大化 + 拆箱便利性

3. **易碎品提醒**
   - 自动标记易碎品（花瓶、玻璃、电子产品）
   - 给出专业包装建议（用衣物包裹、填充气泡膜等）
   - 拍照记录易碎品状态，便于理赔

4. **箱号标签生成**
   - 自动生成二维码标签
   - 扫码查看箱内物品清单
   - 按房间分类（卧室-01、厨房-02）

5. **拆箱指南**
   - 到新家后，按房间推荐拆箱顺序
   - 优先拆箱：洗漱用品、床上用品、常用衣物
   - 提供「房间布置建议」

### 用户流程

```
拍照识别 → 生成清单 → 智能排序 → 打包执行 → 标签打印 → 拆箱指南
```

## 技术方案

### 数据层

- **物品识别**：GPT-4V / GLM-4V / kimi-k2.5 多模态模型
- **物品分类体系**：8 大类（书籍、衣物、电子、厨具、易碎、洗漱、床上、杂物）
- **数据库**：SQLite（本地优先） + 云端同步（可选）
- **存储内容**：
  - 物品清单（名称、类型、数量、位置、照片）
  - 箱子信息（箱号、所属房间、物品列表、二维码）
  - 用户偏好（常用物品、易碎品标记规则）

### 逻辑层

1. **图像识别引擎**
   - 输入：房间照片
   - 处理：物体检测 + 分类 + 计数
   - 输出：结构化物品清单（JSON）

2. **打包排序算法**
   - 规则引擎：基于物品类型的优先级规则
   - 约束优化：箱子容量限制 + 易碎品保护
   - 输出：打包任务队列（每箱包含的物品列表）

3. **标签生成系统**
   - 二维码编码：箱号 + 房间 + 物品摘要
   - 打印格式：A4 纸 10 个标签/页
   - 支持蓝牙打印机直接打印

### 展示层

- **移动端**（优先）：iOS / Android 原生或 Flutter
- **Web 端**：查看清单、导出 PDF
- **交互方式**：
  - 拍照识别（核心交互）
  - 语音补充（「还有 3 本书没拍到」）
  - 手动调整（拖拽排序）

## 实现步骤

### Phase 1（2 周）- MVP 验证

- 拍照识别基础功能（5 个常见物品类别）
- 生成打包清单（文本格式）
- 基础易碎品标记
- 简单的箱子编号

**验收标准**：用户拍照后 30 秒内生成可用的打包清单

### Phase 2（2 周）- 核心体验优化

- 智能打包排序算法
- 二维码标签生成
- 扫码查看箱内物品
- 支持多房间

### Phase 3（2 周）- 拆箱功能

- 拆箱指南生成
- 按房间推荐拆箱顺序
- 提供「房间布置建议」

### Phase 4（2 周）- 增值功能

- 云端同步（多设备）
- 分享清单给家人
- 搬家公司对接（API）

## 资源需求

### API 服务

- **多模态识别**：GPT-4V API 或本地部署 kimi-k2.5
  - 成本：约 ¥0.01-0.05/次识别
  - 优化：本地缓存 + 批量识别

- **云存储**（可选）：阿里云 OSS / 腾讯云 COS
  - 成本：¥0.01/GB/月
  - 用途：照片备份、清单同步

### 第三方服务

- **地图 API**（搬家选址）：高德 / 百度地图
- **打印机 SDK**：蓝牙标签打印机

### 开发资源

- **前端**：1 人（Flutter 或 React Native）
- **后端**：1 人（Node.js + Python）
- **AI 集成**：0.5 人（调用现成 API）
- **设计**：0.5 人（UI/UX）

**总成本估算**：2 个月开发周期，人力成本约 15-20 万

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **识别准确率不足** | 高 - 影响用户信任 | 1. 提供手动补充入口<br>2. 识别置信度提示<br>3. 建立反馈机制持续优化 |
| **物品多样性** | 中 - 奇怪物品识别错误 | 1. 建立「特殊物品」标记机制<br>2. 用户众包标注<br>3. 逐步扩充物品库 |
| **用户习惯** | 中 - 部分用户坚持手写 | 1. AI 辅助 + 人工确认混合模式<br>2. 导入 Excel 功能<br>3. 强调时间节省价值 |
| **数据隐私** | 中 - 家里所有物品被拍照 | 1. 本地处理优先<br>2. 端到端加密<br>3. 明确隐私政策 |
| **竞品模仿** | 低 - 技术门槛不高 | 1. 快速积累用户数据<br>2. 建立搬家场景品牌<br>3. 与搬家公司深度合作 |
