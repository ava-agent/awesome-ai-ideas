# AI 搬家打包指挥官

> **一句话卖点**：租房年轻人在搬家前一周，用 AI 搬家指挥官拍照识别物品，从手忙脚乱打包3天到半天井然有序

## 概述

基于 Issue #165 创建的详细设计文档。

搬家是最让人头疼的琐事之一，现有方案（Excel/备忘录/搬家App）都需要手动输入，太费时间。本产品通过 **AI 辅助识别 + 人工确认** 的混合模式，让用户只需拍照并简单确认即可生成打包清单、智能排序、易碎品提醒，大幅降低搬家焦虑和时间成本。

**核心定位**：AI 辅助而非全自动，降低用户输入成本的同时保证准确性

## 功能设计

### 核心功能

1. **拍照识别物品**（AI 辅助 + 人工确认）
   - 拍摄房间照片，AI 识别物品并给出置信度
   - **混合模式**：AI 自动识别（目标准确率 70%）+ 用户快速确认/补充
   - 支持多张照片拼接，覆盖整个房间
   - 自动识别物品类型：书籍、衣物、电子产品、易碎品等
   - **识别失败降级方案**：引导用户手动标记，或语音输入补充

2. **智能打包顺序**
   - 根据房间、物品类型、使用频率生成最优打包顺序
   - 逻辑：不常用物品（书籍、换季衣物）→ 日常用品 → 易碎品
   - 目标：空间利用率最大化 + 拆箱便利性

3. **易碎品提醒**
   - 自动标记易碎品（花瓶、玻璃、电子产品）
   - 给出专业包装建议（用衣物包裹、填充气泡膜等）
   - 拍照记录易碎品状态，便于理赔

4. **箱号标签生成**
   - 自动生成二维码标签
   - 扫码查看箱内物品清单
   - 按房间分类（卧室-01、厨房-02）

5. **拆箱指南**
   - 到新家后，按房间推荐拆箱顺序
   - 优先拆箱：洗漱用品、床上用品、常用衣物
   - 提供「房间布置建议」

### 用户流程

```
拍照识别 → 生成清单 → 智能排序 → 打包执行 → 标签打印 → 拆箱指南
```

## 技术方案

### 数据层

- **物品识别**：GPT-4V / GLM-4V / kimi-k2.5 多模态模型
- **物品分类体系**：8 大类（书籍、衣物、电子、厨具、易碎、洗漱、床上、杂物）
- **数据库**：SQLite（本地优先） + 云端同步（可选）
- **存储内容**：
  - 物品清单（名称、类型、数量、位置、照片）
  - 箱子信息（箱号、所属房间、物品列表、二维码）
  - 用户偏好（常用物品、易碎品标记规则）

### 逻辑层

1. **图像识别引擎**
   - 输入：房间照片
   - 处理：物体检测 + 分类 + 计数
   - 输出：结构化物品清单（JSON）
   - **置信度机制**：识别置信度 < 70% 的物品标记为"待确认"，引导用户手动确认
   - **分阶段识别**：先识别大件物品（家具、电器）→ 再识别小件物品（书籍、衣物）

2. **打包排序算法**
   - 规则引擎：基于物品类型的优先级规则
   - 约束优化：箱子容量限制 + 易碎品保护
   - 输出：打包任务队列（每箱包含的物品列表）

3. **标签生成系统**
   - 二维码编码：箱号 + 房间 + 物品摘要
   - 打印格式：A4 纸 10 个标签/页
   - 支持蓝牙打印机直接打印

### 展示层

- **MVP 载体**：**微信小程序**（降低获客门槛，无需下载安装）
- **长期规划**：iOS / Android 原生 App（Phase 3 后考虑）
- **Web 端**：查看清单、导出 PDF
- **交互方式**：
  - 拍照识别（核心交互）
  - 语音补充（「还有 3 本书没拍到」）
  - 手动调整（拖拽排序）

| **Excel/备忘录** | 手动记录清单 | 灵活、免费 | 纯手动，无智能功能 | AI 辅助，效率提升 5-10 倍 |

## 实现步骤

### Phase 0（1 周）- 技术可行性验证

**目标**：验证 AI 识别在真实搬家场景的可用性

- 收集 50-100 张真实家庭房间照片（不同光线、堆叠程度）
- 测试 GPT-4V / GLM-4V 识别准确率
- **验收标准**：大件物品识别准确率 > 80%，小件物品 > 60%
- **降级方案**：如准确率不达标，调整为"AI 提示 + 人工快速标记"模式

### Phase 1（3-4 周）- MVP 验证

- 拍照识别基础功能（3 个常见物品类别：书籍、衣物、电子产品）
- 生成打包清单（文本格式）
- 基础易碎品标记
- 简单的箱子编号
- **微信小程序**作为载体（降低获客门槛）

**验收标准**：用户拍照后 30 秒内生成可用的打包清单，人工确认时间 < 2 分钟

### Phase 2（3-4 周）- 核心体验优化

- 智能打包排序算法
- 二维码标签生成（小程序内扫码查看）
- 支持多房间
- 物品依赖关系处理（如：需先拆衣柜才能打包里面的衣服）

### Phase 3（可选，根据用户反馈决定）- 拆箱功能

- 拆箱指南生成
- 按房间推荐拆箱顺序
- **注意**：此功能价值待验证，可能用户不需要

### Phase 4（长期）- 增值功能

- 云端同步（多设备）
- 分享清单给家人
- 搬家公司对接（API）
- 蓝牙标签打印机集成（仅在有明确需求时开发）

## 资源需求

### API 服务

- **多模态识别**：GPT-4V API 或 GLM-4V
  - 单次识别成本：¥0.01-0.05
  - **单次搬家估算**：10-20 张照片，识别 100-200 个物品
  - **单用户单次搬家 AI 成本**：¥0.5-2.0（平均 ¥1.0）
  - **成本优化策略**：
    - 本地缓存已识别物品
    - 批量识别（多张照片合并请求）
    - 用户确认后只对新增物品调用 API

- **云存储**（可选）：阿里云 OSS / 腾讯云 COS
  - 成本：¥0.01/GB/月
  - 用途：照片备份、清单同步

### 第三方服务

- **地图 API**（搬家选址）：高德 / 百度地图
- **打印机 SDK**：蓝牙标签打印机

### 开发资源

- **前端**：1 人（微信小程序）
- **后端**：1 人（Node.js + Python）
- **AI 集成**：0.5 人（调用现成 API）
- **设计**：0.5 人（UI/UX）

**总成本估算**：2 个月开发周期，人力成本约 15-20 万

## 获客策略

### 核心挑战

搬家是低频需求（1-2 年一次），用户获取成本高，需要精准触达：

- **App Store 获客成本**：> ¥10/用户
- **付费转化率**：约 5%
- **单用户获取成本**：> ¥200（unsustainable）

### 解决方案

#### 1. 微信小程序（MVP 载体）

**优势**：
- 无需下载，降低使用门槛
- 社交分享，自然传播
- 获客成本远低于原生 App

**策略**：
- MVP 阶段只做小程序
- 用户搜索"搬家"、"打包"时可发现
- 支持分享清单给家人/室友

#### 2. B2B2C 合作（主要获客渠道）

**搬家公司合作**：
- 用户在搬家时正好有需求，转化率高
- 搬家公司作为增值服务提供给客户
- 采购批量账号，成本 ¥5-10/次

**长租公寓合作**：
- 精准触达租房人群（主要目标用户）
- 预装在公寓 App 中
- 租客入住/退租时使用

#### 3. 内容营销

- 小红书/抖音分享"搬家打包技巧"
- 与家居博主合作推广
- SEO 优化："搬家打包清单"、"搬家注意事项"

### 获客优先级

1. **B2B2C 合作**（搬家公司、长租公寓）- 最高优先级
2. **微信小程序自然搜索** - 中优先级
3. **内容营销** - 低优先级（长期品牌建设）
4. **App Store / 应用市场** - 暂不考虑（成本过高）
