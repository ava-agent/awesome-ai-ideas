# AI 技术会议发言教练

> **一句话卖点**：技术会议沉默者在架构评审会上，用 AI 发言教练收到实时提示，从全程闭麦到抓住时机提出关键问题

## 概述

基于 Issue #125 创建的详细设计文档。

技术人员经常在会议中有想法却不敢说——担心说错显得不专业、不知道如何组织语言、怕打断别人节奏、等想好了话题已经过去。AI 技术会议发言教练帮助技术人员从「全程闭麦」到「主动发言」，通过会议前准备、实时提示、发言演练、渐进式成长四步实现蜕变。

## 功能设计

### 核心功能

1. **会前准备模块**
   - 上传会议议题材料（PPT、文档、代码）
   - AI 生成 3-5 个关键提问角度
   - 提供专业表达模板（STAR、PREP 等结构）
   - 推送今日会议话题卡片

2. **实时提示模块**
   - 会议实时转录（Whisper API / 讯飞语音）
   - 话题追踪与时机识别（停顿、提问、征求意见信号）
   - 推送可发言窗口 + 参考话术
   - 个性化建议（基于用户技术专长）

3. **会后复盘模块**
   - 发言质量分析（结构、清晰度、影响力）
   - 对比「AI 建议话术」vs「实际发言」
   - 未发言想法的模拟对话练习
   - 进步曲线追踪

4. **渐进式成长系统**
   - Level 1：附议型发言（补充他人观点）
   - Level 2：补充型发言（提供额外信息）
   - Level 3：主导型发言（提出新观点）
   - 根据用户水平自动调整建议难度

## 竞品分析与差异化定位

### 主要竞品

| 产品 | 核心功能 | 局限性 | 我们的差异化 |
|------|----------|--------|-------------|
| **otter.ai** | 实时转录 + 会议笔记 | 纯记录工具，无发言指导 | 我们是「发言教练」而非「记录员」 |
| **fireflies.ai** | 会议记录 + 智能搜索 | 聚焦会后总结，无实时干预 | 实时提示 + 发言时机识别 |
| **讯飞听见** | 中文语音转写 | 无发言能力培养 | 渐进式成长系统 + 能力分级 |
| **Grammarly** | 写作辅助 | 仅限文本，无口语场景 | 专注口头表达 + 会议场景 |

### 核心差异化

1. **定位不同**：竞品是「会议记录工具」，我们是「发言能力教练」
2. **实时干预**：不只是记录，而是主动推送发言时机
3. **能力成长**：渐进式系统设计，从 Level 1 到 Level 3
4. **垂直场景**：专注技术会议，而非通用会议

## 技术方案

### 数据层

- **会议数据**：实时语音流 → Whisper API 转写 → 结构化存储
- **用户画像**：技术栈、擅长领域、发言风格偏好
- **话题库**：技术领域常见讨论话题 + 关键词
- **话术模板**：不同场景的表达模板（技术评审、架构讨论、代码评审）

### 逻辑层

- **语音识别引擎**：Whisper API（英文）+ 讯飞语音（中文）
- **话题追踪**：LLM 实时分析会议主题 + 关键词提取
- **时机识别**：检测「停顿」「提问」「征求意见」等信号
- **话术生成**：基于会议内容 + 用户背景 + 技术领域生成个性化建议
- **进步追踪**：多维度评分（结构、清晰度、技术准确性、影响力）

### 展示层

- **桌面客户端**：Electron / Tauri（跨平台）
- **浏览器扩展**：支持 Web 会议（Zoom、腾讯会议 Web 版）
- **移动端**：会议前推送话题卡片，会后查看复盘报告

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **实时性挑战**：语音识别 + LLM 处理延迟可能错过发言窗口 | 提前准备话题卡片，不完全依赖实时生成；优化模型响应速度 |
| **会议隐私**：会议内容可能涉及商业机密 | 提供「本地处理」选项，敏感内容不上云；明确数据保留政策 |
| **企业 IM 集成门槛高**：飞书/钉钉 API 限制 | 先做 Web 会议（Zoom、腾讯会议 Web 版）和本地录制上传 |
| **用户依赖性**：过度依赖 AI 提示 | 设计「渐进式训练」，逐步减少 AI 提示频率；提供离线练习模式 |
| **技术准确性**：话题识别可能出错 | 允许用户手动标注感兴趣话题；持续学习用户偏好 |

## 合规与隐私保护

### 适用法规

- **中国**：《个人信息保护法》（个保法）、《数据安全法》
- **欧盟**：GDPR（如有欧盟用户）
- **美国**：CCPA（如有加州用户）

### 核心合规措施

| 要求 | 实施方案 |
|------|----------|
| **知情同意** | 首次使用时明确告知数据收集范围、用途、保存期限，用户勾选同意 |
| **多方会议** | 会议涉及多方参与者时，提示用户「确保已获得其他参会者同意」 |
| **数据最小化** | 仅收集必要数据（语音转写文本），不存储原始音频（除非用户明确开启） |
| **本地处理选项** | 提供「完全本地模式」，所有语音处理在用户设备完成（使用本地 Whisper 模型） |
| **数据保留** | 默认 30 天自动删除，用户可手动调整或提前删除 |
| **数据导出** | 提供一键导出功能（JSON/Markdown 格式） |
| **账号删除** | 用户可随时注销账号，7 天内彻底删除所有关联数据 |
| **跨境传输** | 如使用海外 API（如 OpenAI），明确告知并获取单独授权 |

### 隐私增强功能

1. **敏感词过滤**：用户可设置关键词（如项目代号、客户名），AI 自动脱敏
2. **匿名化分析**：统计报告不包含具体会议内容，仅保留结构化指标
3. **端到端加密**：传输和存储全程加密（TLS 1.3 + AES-256）
4. **第三方审计**：定期进行安全审计，公开审计报告

## 关键指标

- **北极星指标**：用户发言次数变化（会议前 vs 会议后）
- **参与度指标**：每周使用会议数、会前准备完成率
- **效果指标**：用户自评发言信心提升、同事反馈改善率
- **商业指标**：付费转化率、月留存率
