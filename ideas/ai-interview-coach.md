# AI 面试场景模拟教练

> **一句话卖点**：求职者在面试前用 AI 模拟真实对话场景，从紧张卡壳到从容应答，提升面试通过率

## 概述

基于 Issue #40 创建的详细设计文档。

求职面试是职场人最焦虑的场景之一。背了无数面经，但一开口就紧张；想找人模拟面试，但真人教练太贵或不好意思麻烦朋友。AI 面试模拟教练通过语音对话模拟真实面试场景，帮助求职者克服紧张、提升表达、从容应对各种追问。

## 功能设计

### 核心功能

1. **智能简历解析**
   - OCR 识别简历图片/PDF
   - 自动提取工作经历、技能亮点、教育背景
   - 识别可能的追问点（如频繁跳槽、职业空白期）

2. **岗位适配题库**
   - 根据职位描述（JD）生成针对性问题
   - 支持多岗位类型：技术、产品、运营、管理等
   - 难度分级：入门、中级、高级

3. **语音模拟对话**
   - 语音输入回答，模拟真实面试场景
   - AI 面试官根据回答实时追问
   - 支持压力面试模式（追问棘手问题）

4. **实时分析反馈**
   - 评估回答的逻辑性、完整性、表达流畅度
   - 识别口头禅、重复表达等问题
   - 给出改进建议和参考答案

5. **面试复盘报告**
   - 综合评分（逻辑、表达、专业度）
   - 薄弱环节分析
   - 定制化练习建议

### 辅助功能

- 历史模拟记录
- 面试题库浏览
- 行业薪资参考
- 面试经验分享社区

## 技术方案

### 数据层

```
├── 用户数据
│   ├── 简历文档（OSS 存储）
│   ├── 面试历史记录
│   └── 个性化设置
├── 题库数据
│   ├── 岗位题库（按行业/岗位分类）
│   ├── 追问模板库
│   └── 参考答案库
└── 评估数据
    ├── 表达评分模型
    ├── 逻辑分析规则
    └── 行业标准答案
```

### 逻辑层

**核心流程：**

```
简历上传 → OCR 解析 → 关键信息提取
    ↓
JD 输入 → 岗位识别 → 题目匹配
    ↓
语音对话 → ASR 转文字 → LLM 分析 → TTS 反馈
    ↓
综合评估 → 生成报告 → 推荐练习
```

**技术栈：**

- **ASR**: Whisper（本地或 API）
- **OCR**: 百度/腾讯云 OCR（简历识别）
- **LLM**: GLM-4 / Claude Haiku（性价比高）
- **TTS**: Edge TTS（免费）或 Azure TTS（高质量）
- **后端**: Node.js + Express
- **前端**: React + Web Speech API

### 展示层

**移动端优先：**
- 录音界面（大按钮，操作简单）
- 实时文字反馈
- 面试报告（可视化图表）
- 题库浏览（卡片式）

**Web 端补充：**
- 简历上传与管理
- 详细报告查看
- 社区交流

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 冷启动题库不足 | 用户体验差 | 先聚焦 2-3 个热门岗位，逐步扩展；用户可上传题目 |
| 语音延迟过高 | 用户流失 | 1) 优化 pipeline（流式处理）<br>2) 提供"低延迟模式"（简化追问）<br>3) 保留文字版作为备选 |
| 用户付费意愿低 | 商业化困难 | 1) 免费版验证需求<br>2) B2B 路线（高校、培训机构）<br>3) 增值服务（简历优化、真人教练） |
| 评估准确性争议 | 用户信任下降 | 1) 明确标注"仅供参考"<br>2) 引入行业标准答案<br>3) 结合真人教练审核 |
| 简历隐私泄露 | 法律风险 | 1) 本地处理敏感信息<br>2) 加密存储<br>3) 明确隐私政策，用户授权 |
