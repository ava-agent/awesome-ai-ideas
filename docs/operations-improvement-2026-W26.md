# 运营改进建议 2026-W26

## 1. 自动化简化

**现状**：每2小时生成repo pulse、quality snapshot等报告，导致docs/automation/下104个文件，大多是重复信息。

**改进**：
- 将频率从2小时调整为**每日1次**
- 合并相似报告类型，只保留关键指标
- 添加自动清理机制，保留最近7天

## 2. 高价值idea推进机制

**现状**：idea-tracker.json中标记为high-value的idea停留在文档阶段，缺乏实际推进。

**改进**：
- 为每个high-value idea设定**月度小目标**（访谈1个用户、写1页MVP spec等）
- 建立"idea→验证→MVP"的明确里程碑追踪
- 每季度**归档或升级**停滞的idea

## 3. 社区互动闭环

**现状**：只有单向内容输出，没有外部反馈机制。

**改进**：
- README添加简单的**投票/反馈链接**（可使用GitHub Discussions）
- 每个idea文件添加"讨论区"section链接
- 每月汇总1次社区信号到周报
