# AI 穿搭决策顾问

> **一句话卖点**：每天早上在衣柜前纠结 10-20 分钟的职场女性，拍照说需求，3 秒获得 3 套基于真实衣柜的搭配方案 + 理由，从「试了5套还是不满意」到「自信出门」

## 概述

基于 Issue #178 创建。穿搭纠结是职场人的普遍痛点 —— 每天花 10-20 分钟选衣服，一年累计 60-120 小时，相当于 **15 个工作日** 被浪费在决策疲劳上。更糟糕的是，买了新衣服却闲置在柜子里，因为不知道怎么搭配；小红书搜穿搭看完更纠结，信息过载；问朋友不一定在线，穿搭 App 只展示模特图和你的衣柜无关。

本产品定位「**决策型**穿搭工具」，区别于现有「展示型」穿搭 App。核心路径：**拍照 + 说需求 → AI 分析天气/场合/衣柜 → 3 套可执行方案 + 搭配理由**。

**产品经理评分**：4.2/5（需求真实，冷启动是关键卡点）
**增长黑客评分**：3.8/5（社交裂变潜力大，但衣柜数字化阻力高）
**技术专家评分**：8/10（多模态+推荐技术成熟，关键在数据冷启动）

**核心价值**：不是给你看别人怎么穿，而是基于你的真实衣柜，帮你做决策，并且越用越懂你。

## 竞品分析

### 主要竞争对手

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|

| **搭搭/穿衣助手** | 穿搭社区 | 用户量大 | 手动记录为主，AI 能力弱 | 多模态识别自动分类，场景感知推荐 |

| **Stylebook** | 衣柜管理 | 衣柜数字化成熟 | 无推荐能力，纯工具 | 管理智能推荐一体化 |
| **淘宝/京东穿搭** | 电商导购 | 商品库丰富 | 推荐你买更多，不是帮你搭已有的 | 只推荐你有的单品，不制造消费焦虑 |

### 护城河分析
1. **中国本土化**：接入淘宝/京东购买记录自动导入、中国审美体系（职场正式度分级）、中国气候分区穿搭建议
2. **决策型 vs 展示型**：不只给你灵感，直接给你「今天穿什么」的答案
3. **衣柜资产壁垒**：用户上传 50+ 件衣服后迁移成本极高

## 功能设计

### 核心功能

**1. 衣柜数字化（渐进式）**
- 拍照上传 → AI 自动识别款式/颜色/材质/风格标签
- 分类体系：上装/下装/外套/鞋/配饰，再按风格（正式/休闲/运动）细分
- 支持淘宝/京东订单导入（OCR 识别订单截图 + 自动匹配商品图）
- 隐私优先：原图本地存储，云端仅存特征向量（CLIP embedding）

**2. 场景感知推荐引擎**
- 输入维度：天气（温度+降水+风力）、场合（会议/约会/面试/日常）、心情（正式/活力/舒适）、日程
- 输出：3 套方案，每套标注搭配理由 + 适用场景 + 上次穿着反馈
- 支持自然语言输入：「今天有汇报，外面下雨，想显得专业但不沉闷」

**天气适配细节**：
- 室内外差异：空调房 vs 户外通勤（支持设置通勤方式：地铁/骑车/开车）
- 个人温度偏好：用户可设置「我怕冷」「我怕热」
- 天气 API：和风天气（支持实时 + 预报）

**日历集成（可选）**：
- 支持平台：Apple Calendar、Google Calendar、Outlook
- 隐私保护：需用户明确授权，仅读取事件标题/时间，不读取详情
- 场景推断：会议→正式、约会→浪漫、运动→休闲

**3. 个性化学习系统（详细设计）**

**学习算法**：
- **协同过滤**：找到风格相似的用户，推荐他们喜欢的搭配
- **内容推荐**：基于颜色、风格、场合的向量相似度匹配
- **混合权重**：初期 80% 规则 + 20% 协同过滤 → 稳定后 50% + 50%

**数据需求**：
- 冷启动：5 件衣服 + 3 次选择反馈即可开始个性化
- 稳定推荐：15+ 件衣服 + 10+ 次穿着记录
- 深度学习：30+ 件 + 30+ 次记录，Pro 版专属

**风格变化适应**：
- 季节性自动调整（冬季偏好深色/保暖 → 夏季偏好浅色/透气）
- 生活事件标记（换工作/怀孕/搬家）触发重新学习
- "风格偏好仪表盘"：用户可查看 AI 学到的偏好并手动调整

**核心功能**：
- 记录每次推荐的选择和反馈（穿了/没穿/原因）
- 学习用户偏好：颜色倾向、风格偏好、回避单品
- 发现隐藏模式：「穿蓝色系那天汇报反馈好」「白色衬衫你选了 80% 的时间」
- **风格档案仪表盘**：展示 AI 学到的偏好（颜色分布、风格倾向、常选单品），用户可手动调整

**4. 新衣搭配建议**
- 拍新衣服 → 立刻匹配衣柜中可搭配的 3-5 件单品
- 「买这件可以搭出 N 套新 look」—— 解决新衣闲置问题
- 购买建议评分：「你衣柜里已有 3 件类似的，建议不买」

**5. 穿搭心情日记（彩蛋功能）**
- 记录每天穿搭 + 当天心情/事件
- 长期积累后 AI 发现「穿暖色系那天效率更高」等隐藏关联
- 月度穿搭报告：最常穿单品、穿搭多样性评分、衣橱利用率

**6. 新功能（根据 Review 反馈新增）**

**🎲 穿搭轮盘（Outfit Roulette）**
- 一键随机推荐衣柜中的搭配
- 游戏化元素，帮助用户发现被遗忘的单品
- 新用户立即体验价值，降低冷启动阻力

**🌧️ 天气预警推送**
- "明天有雨！为你准备了 3 套雨天友好搭配"
- 主动展示价值，增加每日互动
- 前一天晚上推送，提前准备

**📦 精简衣橱生成器（Capsule Wardrobe Generator）**
- 选择 N 件单品（10/20/30），最大化搭配组合数
- 适用场景：出差旅行打包、季节换装
- 展示 AI 的战略思考能力

**👕 洗衣周期追踪**
- 标记单品状态：「可穿」「在洗衣篮」「晾晒中」
- 更实用的日常推荐，避免推荐已穿脏的衣服
- 增加每日使用频次

### 差异化优势

| 现有方案 | 本产品 |
|---------|--------|
| 小红书刷 30 分钟穿搭攻略，看完更纠结 | 拍照说需求，3 秒给 3 套方案 + 理由 |
| 买新衣服拍给朋友问「怎么搭」，朋友不一定在线 | 拍照立即匹配衣柜里 3-5 件可搭配单品 |
| 穿搭 App 只展示模特图，和你衣柜无关 | 基于真实衣柜推荐，不推荐你没有的 |
| 每次决策都是新的纠结，没有进步 | AI 记录偏好，越用越懂你的风格 |
| ChatGPT 问穿搭，每次从零开始 | 持久化衣柜 + 偏好档案，越用越准 |

## 用户冷启动策略

### 最大挑战：衣柜数字化阻力
用户懒得一件件拍照上传，这是生死线。

### 解决方案：渐进式 + 自动导入

**Step 1：零门槛入场（0 分钟）**
- 新用户无需上传任何衣服
- 选择「我的风格」标签（简约/职场/休闲/潮流）
- 立即获得基于通用衣橱的推荐，体验核心价值

**Step 2：快速画像（3 分钟）**
- 引导拍 5 件最常穿的衣服（贴纸引导：正面平铺/挂拍）
- AI 识别 → 建立基础风格画像
- 此时推荐准确度约 50-60%

**Step 3：淘宝/京东导入（1 分钟）**
- 上传订单截图或授权登录
- OCR 识别 + 商品图匹配 → 自动导入购买记录
- 一次导入 20-50 件，准确度提升到 70-80%

**Step 4：持续完善**
- 每次使用时「今天穿的是这套吗？」快速标注
- 日常穿着自然积累，2 周后准确度 85%+

### 留存机制
- **每日穿搭提醒**：早上 7:00 推送「今天 15°C 有雨，已为你准备 3 套方案」
- **穿搭成就**：连续 7 天不重复、衣橱利用率 > 60%、月度穿搭多样性勋章
- **社交分享**：「今日穿搭」卡片分享到小红书/朋友圈（自然引流）
- **周报**：本周穿了什么、衣橱死角提醒（30 天没穿的单品）

## 技术方案

### 数据层
- **衣柜数据**：服装特征向量（CLIP embedding 512d）、分类标签、用户标注
- **用户画像**：风格偏好向量、颜色倾向、场景-穿搭映射、穿着历史
- **外部数据**：天气 API（和风天气）、日历集成（读取日程判断场合）
- **存储策略**：原图本地（手机），特征向量云端（PostgreSQL + pgvector）

### 逻辑层

```
用户输入（拍照 + 文字需求）
        ↓
┌───────────────────────┐
│  场景解析引擎          │
│  - 天气 API → 温度/降水 │
│  - 日历 → 场景类型      │
│  - NLU → 风格/心情/偏好 │
└───────────┬───────────┘
            ↓
┌───────────────────────┐
│  搭配推荐引擎          │
│  - 衣柜检索（向量相似度）│
│  - 搭配规则（LLM 生成） │
│  - 个性化排序          │
└───────────┬───────────┘
            ↓
    3 套方案 + 理由 + 评分
```

**搭配规则知识库**（LLM 初始化 + 用户反馈迭代）：
- 颜色搭配规则（互补色/同类色/三色法则）
- 场景-正式度映射（面试 90% / 日常 30% / 约会 60%）
- 材质搭配禁忌（丝绒+牛仔、皮衣+运动裤）
- 中国职场穿搭规范（金融/互联网/国企差异）

**推荐算法**：
- 冷启动：基于风格标签 + 搭配规则的规则引擎
- 有数据后：协同过滤（相似用户穿搭）+ 内容推荐（向量检索）混合
- 个性化权重：用户选择频率 × 时间衰减 × 场景匹配度

### 展示层
- **移动端 App**（核心）：拍照 + 对话式交互，早上使用场景
- **微信小程序**：轻量版，降低下载门槛，适合冷启动
- **Web 端**：衣柜管理 + 穿搭报告（详细查看用）

### 技术栈
- **服装识别**：CLIP ViT-L/14（零样本分类）+ LoRA 微调（服装风格）
- **搭配推荐**：GLM-4 / Qwen-Max（生成搭配理由）+ 规则引擎
- **向量检索**：pgvector（PostgreSQL 扩展），HNSW 索引
- **前端**：React Native（App）+ Taro（小程序）
- **后端**：Node.js / Python FastAPI + PostgreSQL + Redis
- **天气**：和风天气 API（免费版 1000 次/天）
- **OCR**：PaddleOCR（订单截图识别，开源免费）

### 隐私架构
```
手机本地存储：          云端存储：
- 原始照片             - CLIP embedding 向量（不可逆）
- 服装标签             - 用户偏好数据（加密）
- 穿着历史             - 推荐记录
- 本地推荐缓存         - 匿名统计（可选）
```

## 目标用户

### 核心用户画像
- **职场人士 22-35 岁**：每天有穿搭需求，场合多样（会议/汇报/团建/约会）
- **穿搭纠结族**：衣柜满满但每天不知道穿什么
- **冲动购物者**：买了很多但不搭配，衣橱利用率 < 40%
- **注意**：产品定位为性别中立，从 Day 1 即支持男女用户（男性同样有穿搭困扰，且竞争更少）

### 扩展用户
- **大学生**：面试/约会/社交场景穿搭
- **时尚新手**：不知道怎么搭配，想学习穿搭知识
- **男性用户**：职场男士穿搭（市场需求大，竞争更少，从 Day 1 支持）

### 用户分层运营
| 层级 | 占比 | 特征 | 策略 |
|------|------|------|------|

| 活跃免费 | 20% | 每日用，衣柜 15-30 件 | 个性化推荐展示 Pro 版价值 |

## 竞争壁垒

1. **衣柜数据壁垒**：用户上传 30+ 件衣服后迁移成本极高（重新拍照 = 重新开始）
2. **个性化模型**：积累的用户偏好数据让推荐越来越准，新玩家无法复制
3. **搭配知识库**：LLM 生成 + 用户反馈迭代，形成独特的搭配规则体系
4. **中国本土化**：淘宝/京东导入、中国职场穿搭规范、中国气候适配
5. **习惯养成**：每日穿搭推荐成为用户晨间习惯后，替换成本极高
6. **网络效应**：匿名聚合的穿搭数据 → 更好的搭配建议 → 更多用户
