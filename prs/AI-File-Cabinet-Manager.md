# AI 文件柜管家 - 智能纸质文档管理方案

> **一句话卖点**：纸质文档囤积族在办签证/房产交易时，用 AI 文件柜管家拍照归档，从翻箱倒柜2小时到3秒搜出任何文件

## 概述

基于 Issue #217 创建的详细设计文档。

**目标用户**：有大量纸质文档（证件、合同、发票、证书）需要管理的个人用户

**核心痛点**：
- 纸质文档无处安放，塞得到处都是
- 需要时找不到，翻箱倒柜浪费时间
- 不知道该保留什么、可以扔什么
- 数字化工具要手动分类，太麻烦

**解决方案**：拍照即归档的智能文档管理系统，AI 自动识别、分类、标签，支持全文搜索和保留建议。

## 功能设计

### 核心功能

1. **拍照即归档**
   - 支持相机拍照或相册导入
   - AI 自动识别文档类型（证件/合同/发票/证书/银行卡等 15+ 类别）
   - 自动裁剪、增强图像质量
   - 批量拍照支持，一次处理多份文档

2. **智能标签与搜索**
   - 自动提取关键信息（日期、机构、金额、有效期）
   - 支持全文 OCR 搜索
   - 标签系统：类型标签 + 自定义标签
   - 智能联想搜索（"社保" → "2023年社保缴纳证明"）

3. **保留建议**
   - 基于文档类型和法律要求，智能建议保留期限
   - 到期提醒：提前 30 天提醒文档即将过期/可销毁
   - 保留规则库（身份证 10 年、发票 5 年、合同到期后 3 年等）

4. **安全存储**
   - 本地加密 + 云端加密双重保护
   - 生物识别解锁（Face ID / Touch ID）
   - 敏感文档支持仅本地存储模式
   - 导出带水印，防止滥用

5. **一键导出**
   - 办事时一键导出所需文档的电子版
   - 支持 PDF / 图片格式
   - 按用途生成文档包（签证材料、房产交易、入职材料）

### 辅助功能

- 文档过期提醒
- 重要日期标记
- 文档分享（加密链接）
- 多设备同步

## 技术方案

### 数据层

**本地存储**：
- SQLite + SQLCipher 加密
- 敏感文档的图像存储在本地加密区域
- 元数据（标签、日期、类型）本地缓存

**云端存储**：
- 对象存储（OSS / S3）
- 服务器端加密（SSE-S3）
- 用户隔离（每个用户独立存储空间）

**数据同步**：
- 增量同步策略
- 冲突解决：最后修改优先
- 离线优先设计

### 逻辑层

**文档识别**：
- OCR：百度 OCR API / 腾讯云 OCR
- 文档分类：CLIP + 规则引擎
- 关键信息提取：NER（命名实体识别）

**保留规则引擎**：
- 规则数据库（文档类型 → 保留期限）
- 法律合规检查
- 动态更新规则库

**安全模块**：
- AES-256 加密
- 密钥管理：用户密码派生 + 服务器辅助
- 访问控制：基于角色的权限管理

### 展示层

**移动端（MVP）**：
- iOS / Android 原生应用
- SwiftUI / Jetpack Compose
- 相机集成：AVFoundation / CameraX

**Web 端（后续）**：
- React + TypeScript
- 响应式设计
- PWA 支持

**关键界面**：
- 首页：最近文档 + 快速搜索
- 拍照页：相机 + 批量模式
- 文档详情：预览 + 标签 + 操作
- 搜索页：全文搜索 + 筛选器

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **隐私安全**：用户担心敏感文档泄露 | 1. 本地加密 + 云端加密双重保护<br>2. 敏感文档支持仅本地存储<br>3. 通过 ISO 27001 认证<br>4. 透明的隐私政策 |
| **OCR 准确率**：照片质量参差影响识别 | 1. 图像预处理（去噪、增强）<br>2. 多次识别取最佳结果<br>3. 用户手动修正机制 |
| **低频需求**：用户一年只用几次 | 1. 从高价值场景切入（签证、房产）<br>2. 扩展到租房场景（频次更高）<br>3. 增加日常文档管理（发票、保修卡） |
| **竞品压力**：扫描全能王、腾讯文档 | 1. 差异化：自动分类 + 保留建议<br>2. 专注个人文档管理，不做企业<br>3. 更简洁的 UX，零学习成本 |
| **数据冷启动**：新用户无历史数据 | 1. 批量导入功能<br>2. 预置常见文档模板<br>3. 智能推荐保留规则 |
