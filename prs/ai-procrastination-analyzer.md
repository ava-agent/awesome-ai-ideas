# AI 拖延模式分析师

> **一句话卖点**：拖延族在凌晨懊恼时，用 AI 分析师找到根因，3天从"明天一定改"的无限循环中解脱出来

## 概述

基于 Issue #166 创建的详细设计文档。

拖延不是懒，也不是时间管理差，而是心理机制在作祟。现有工具（番茄钟、待办清单）只管"怎么做"，不分析"为什么不做"。本产品通过记录拖延发生的时间、任务类型、情绪状态，用 AI 识别个人的拖延类型，提供科学验证的针对性对策。

**核心价值**：识别根因 + 对症下药，而不是强制自律。

## 功能设计

### 核心功能

1. **模式追踪引擎**
   - 记录维度：时间（深浅？）、任务类型（创意/执行？）、情绪状态（焦虑/无聊？）、前序事件（刚被批评？）
   - 一键记录：拖拽式输入，3 秒完成
   - 自动关联：智能识别拖延前 5 分钟的行为链

2. **拖延类型分类器**
   - 完美主义型：害怕做得不够好 → 永远不开始
   - 恐惧失败型：担心结果不如预期 → 用拖延保护自尊
   - 任务厌恶型：任务太无聊/太难 → 大脑寻求即时 gratification
   - 注意力分散型：容易被新事物吸引 → 专注时间碎片化
   - 置信度评分：让用户知道「我 85% 确定你是完美主义型」

3. **个性化方案推荐**
   - 规则引擎：每种拖延类型对应科学验证的对策
     - 完美主义 → 「5 分钟粗糙开始法」
     - 恐惧失败 → 「小步快跑 + 正向反馈」
     - 任务厌恶 → 「番茄工作法 + 奖励机制」
     - 注意力分散 → 「环境隔离 + 专注模式」
   - AI 优化：基于用户反馈持续调整方案

4. **预防预警系统**
   - 高危场景识别：周五下午 + 复杂任务 = 高风险
   - 智能推送：在拖延发生前 5 分钟干预

5. **进展追踪仪表盘**
   - 可视化：拖延模式变化趋势图
   - 里程碑：看到真实进步（拖延次数减少 30%）
   - 周报：每周生成个人拖延分析报告

## 技术方案

### 数据层
- **时间序列数据库**：TimescaleDB（存储追踪数据）
- **埋点 SDK**：轻量级，支持 Web/iOS/Android
- **数据脱敏**：本地预处理 + 端到端加密

### 逻辑层
- **分类器**：决策树 + 轻量级 ML（K-Means/Random Forest）
- **规则引擎**：每种拖延类型 → 对策库映射
- **AI 增强**：LLM（如 kimi-k2.5）生成个性化建议

### 展示层
- **前端**：React Native（跨平台）+ Recharts（可视化）
- **推送**：FCM/APNs 智能干预
- **离线支持**：本地缓存 + 后台同步

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 效果验证难 | 设计客观指标（拖延时间减少比例），而非主观感受 |
| 用户过度依赖 | 设计「渐进式放手」机制，培养自我觉察能力 |
| 隐私敏感（情绪数据） | 本地处理优先 + 端到端加密 + 用户完全控制 |
| 竞品分流（Todoist/Forest） | 差异化定位：分析「为什么不做」而非「怎么做」 |
| 用户留存低 | 聚焦单一类型（完美主义型）先验证，这类付费意愿最强 |

## 差异化优势

| 现有方案 | 本产品 |
|---------|--------|
| 番茄钟：强制专注，不解决心理障碍 | 分析为什么不想开始，针对性破解 |
| 待办清单：列一堆任务，越看越焦虑 | 识别任务类型与情绪关联 |
| 时间追踪：只记录时间花在哪 | 追踪拖延前的行为模式，发现触发链条 |
| 自律课程：通用方法，听过就忘 | 基于个人数据，持续优化方案 |
| 心理咨询：有效但贵且慢 | 低成本、随时可用、基于真实行为数据 |

## 市场规模

- **目标人群**：全球约 20% 成年人有慢性拖延问题（约 16 亿人）
- **付费意愿**：用户愿意为「摆脱自我厌恶」付费
- **市场空白**：Todoist/Forest 只解决「怎么做」，不解决「为什么不做」
