# AI 桌游规则翻译官

> **一句话卖点**：桌游新手在聚会前10分钟，用 AI 规则翻译官拍照10秒获得3句核心玩法，从听规则3遍还懵到看一眼就会玩

## 概述

基于 Issue #192 创建的详细设计文档。

**目标用户**：桌游新手、聚会组织者、规则讲解困难户
**核心痛点**：规则书晦涩难懂、聚会前10-15分钟都在等规则讲解、怕问蠢问题假装懂了
**解决方案**：AI 驱动的桌游规则翻译系统，从拍照识别到3句话讲清楚核心玩法

## 功能设计

### 核心功能

1. **拍照识规则**
   - 拍规则书/盒子封面，AI 10秒提取核心玩法
   - 支持中英文规则书识别
   - 自动识别游戏类型（策略/聚会/推理）

2. **3句话讲清楚**
   - 目标是什么、怎么行动、怎么赢——就这三句
   - 去除所有专业术语和冗余描述
   - 配合简单图示加深理解

3. **回合演示**
   - AI 用动画/图示模拟一个完整回合
   - 可视化展示每个玩家的行动
   - 支持暂停/重播/慢放

4. **语音答疑**
   - 玩的过程中随时问「这个能这样用吗？」
   - AI 实时回答，不尴尬
   - 支持追问和澄清

5. **复杂度适配**
   - 根据玩家水平调整解释详细程度
   - 新手模式：超详细，每个概念都解释
   - 老手模式：只讲关键差异和特殊规则

## 竞品分析

### 主要竞品

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
| **B站视频教程** | 视频讲解 | 内容丰富、真人演示 | 15分钟起步、聚会时没人愿意看 | 拍照10秒、3句话讲完 |
| **BoardGameGeek** | 全球桌游社区 | 规则库庞大、讨论活跃 | 全英文、需要主动搜索 | 中文原生、拍照即用 |
| **桌游吧老板教** | 线下服务 | 真人互动、即时答疑 | 需要到店、老板可能不熟悉新游戏 | 随时随地、覆盖所有游戏 |
| **规则书** | 官方文档 | 权威完整 | 文字密密麻麻、术语劝退 | 3句话总结、可视化演示 |

### 我们的差异化优势

1. **速度优势**：10秒拍照 → 3句话核心玩法，比视频教程快10倍
2. **零门槛**：不需要提前学习，聚会现场即扫即用
3. **不尴尬**：问 AI 不怕被嫌弃，随时追问
4. **可视化**：动画演示一个回合，看一眼就懂
5. **适配性**：根据玩家水平自动调整详细程度

## 技术方案

### 数据层

**游戏数据库**
- 游戏元数据：名称、类型、人数、时长、难度
- 规则结构：目标、行动、胜利条件
- 常见问题库：玩家高频问题及解答

**用户数据**
- 游戏记录：玩过的游戏、偏好类型
- 学习进度：掌握的游戏规则
- 反馈数据：规则解释质量评分

**存储方案**
- 游戏元数据：PostgreSQL
- 规则向量：Qdrant（开源向量数据库）
- 用户数据：PostgreSQL + Redis

### 逻辑层

**规则识别服务**
```
输入：规则书照片/扫描件
处理：
  1. OCR 识别 → 提取文字
  2. 结构化解析 → 目标/行动/胜利条件
  3. 术语简化 → 转换为通俗语言
  4. 向量化 → 存入向量库
输出：结构化规则数据
```

**3句话生成服务**
```
输入：结构化规则 + 用户水平
处理：
  1. 目标提取 → 一句话说明游戏目标
  2. 行动概括 → 一句话说明玩家能做什么
  3. 胜利条件 → 一句话说明怎么赢
  4. 复杂度调整 → 根据用户水平增减细节
输出：3句核心玩法 + 可视化建议
```

**问答服务**
```
输入：用户问题 + 当前游戏上下文
处理：
  1. 问题理解 → 识别用户问的是什么
  2. 规则检索 → 从向量库中找相关规则
  3. 答案生成 → 用通俗语言回答
  4. 示例补充 → 如果需要，补充实际例子
输出：精准回答 + 相关规则引用
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 规则书格式多样 | 先支持主流格式，逐步扩展；用户可反馈识别错误 |
| 3句话不够清晰 | 提供「展开详情」选项，支持查看完整规则 |
| 冷启动困难 | 预置50款热门游戏规则，新用户可先体验再拍照 |
| 版权问题 | 只处理用户购买的规则书，不提供规则分享 |
| 用户不习惯拍照 | 提供手动输入游戏名称的方式 |
| 回答不准确 | 用户可反馈纠正，持续优化模型 |
