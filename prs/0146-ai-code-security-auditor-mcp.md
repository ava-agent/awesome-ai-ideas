# AI Code Security Auditor MCP Server — AI 代码安全审计 MCP 服务器

> **一句话卖点**：用 Claude Code/Cursor 写完代码，一句话调用 MCP 工具扫描 AI 生成代码的安全漏洞，3 秒出报告+修复建议，比手动审计快 50 倍

## 概述

基于 Issue #146 及评论讨论整合。Dev.to 社区热议揭示两个关键趋势：

1. **MCP（Model Context Protocol）正成为 AI Agent 的标准接口** — Claude Code、Cursor、Windsurf 等主流 AI IDE 均已原生支持 MCP，生态处于爆发期（2025 Q1 MCP Server 数量环比增长 340%）
2. **AI 生成代码的安全问题被严重忽视** — 研究显示 100 个 AI 生成应用中 30%+ 存在安全漏洞，典型问题包括硬编码密钥（34%）、SQL 注入（28%）、XSS（22%）

本产品定位为 **MCP 原生的 AI 代码安全审计工具**，将安全扫描嵌入 AI 编码工作流，而非事后检查。开发者在使用 Claude Code/Cursor 生成代码时，可直接调用 `security-audit` 工具，获得上下文感知的漏洞检测和修复建议。

**核心价值**：不是又一个 SAST 工具，而是专门针对 AI 生成代码漏洞模式优化的、零配置即插即用的 MCP 安全服务器。

### 我们的护城河
1. **AI 代码漏洞模式库** — 针对 AI 生成代码的特有漏洞模式（硬编码密钥、不安全默认值、缺失输入验证、过时依赖版本）持续积累
2. **MCP 原生** — 嵌入 Claude Code/Cursor 工作流，不需要切换工具
3. **上下文感知** — 理解代码意图，误报率比通用 SAST 工具低 40%+
4. **开源社区** — 规则库由社区贡献，快速覆盖新漏洞类型

## 功能设计

### 核心功能
1. **实时安全扫描** — 作为 MCP 工具，可被 Claude Code/Cursor 等直接调用，`npx ai-security-scan` 一键扫描
2. **AI 代码模式识别** — 针对 AI 生成代码的典型漏洞模式：
   - 硬编码密钥/API Key（检测率目标 95%+）
   - SQL 注入 / NoSQL 注入
   - XSS（反射型 + 存储型）
   - 不安全依赖（已知 CVE）
   - 缺失输入验证
   - 不安全的默认配置
3. **上下文感知审计** — 理解代码意图，区分「硬编码密钥」（漏洞）和「测试 fixture」（误报），误报率目标 < 10%
4. **修复建议生成** — 不只报错，还给出可直接应用的修复代码（通过 MCP 返回 patch）
5. **CI/CD 集成** — GitHub Actions 插件，PR 自动扫描

## 技术方案

### 架构设计（三层）

```
┌─────────────────────────────────────────────────┐
│                   展示层（MCP 接口）               │
│  MCP Tools: scan_code, scan_project, fix_issue  │
│  CLI: npx ai-security-scan                      │
│  CI/CD: GitHub Actions Plugin                    │
├─────────────────────────────────────────────────┤
│                   逻辑层（扫描引擎）               │
│  第一层: Semgrep 规则快速扫描（已知漏洞，< 2s）    │
│  第二层: AI 模型识别 AI 特有模式（硬编码密钥等）   │
│  第三层: 上下文分析降误报（LLM 理解代码意图）      │
├─────────────────────────────────────────────────┤
│                   数据层                         │
│  漏洞规则库（YAML + AI Pattern）                  │
│  CVE 数据库（npm audit / osv.dev）               │
│  项目扫描缓存（SQLite，增量扫描）                 │
└─────────────────────────────────────────────────┘
```

### 三层扫描引擎详解

| 层级 | 引擎 | 扫描内容 | 耗时 | 误报率 |
|------|------|---------|------|--------|
| **L1 快速扫描** | Semgrep（内置 50+ 规则） | SQL 注入、XSS、路径遍历、命令注入等 OWASP Top 10 | < 2s（单文件） | 15-20% |
| **L2 AI 模式识别** | 本地轻量模型 + 规则引擎 | 硬编码密钥、不安全默认值、过时依赖、缺失验证 | < 5s（单文件） | 10-15% |
| **L3 上下文分析** | LLM API（可选，需配置 API Key） | 代码意图理解，区分真漏洞与测试代码 | < 10s（单文件） | < 5% |

- **默认模式**（免费）：L1 + L2，覆盖 80%+ 常见漏洞
- **深度模式**（Pro）：L1 + L2 + L3，上下文感知，误报率最低

### 技术栈

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| MCP Server | TypeScript + @modelcontextprotocol/sdk | 官方 SDK，生态最成熟 |
| 静态分析 | Semgrep（内置） | 开源免费，规则灵活，AST 级分析 |
| 依赖扫描 | osv.dev API + npm audit | 免费开源漏洞数据库 |
| AI 模式识别 | tree-sitter（AST 解析）+ 自定义规则 | 轻量，无需 GPU，本地运行 |
| 上下文分析（L3） | LLM API（Claude/GPT-4o-mini，用户自带 Key） | 按需调用，不内置 API Key |
| 缓存 | SQLite（本地） | 零依赖，增量扫描，快 |
| CLI | commander.js + chalk | Node.js 生态标准 |
| CI/CD | GitHub Actions composite action | 原生 GitHub 集成 |
| 包管理 | npm + npx | 开发者零安装体验 |
