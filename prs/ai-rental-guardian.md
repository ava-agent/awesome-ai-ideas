# AI 租房避坑顾问

> **一句话卖点**：租房小白在签约前 5 分钟，用 AI 租房避坑顾问扫描合同，从被动挨坑到主动避雷

## 概述

基于 Issue #176 创建的详细设计文档。

每年超过 100 万起租房纠纷案件，70% 涉及合同陷阱。AI 租房避坑顾问为租房族提供全流程护航：拍照识别合同风险、查询片区租金参考、生成维权话术。让租房小白从"任人宰割"到"明明白白租房"。

## 功能设计

### 核心功能

1. **合同秒扫**
   - 拍照/上传合同关键页
   - OCR + LLM 识别风险条款
   - 高亮标注 + 风险解释 + 改写建议
   - 支持押金条款、违约责任、维修义务、提前解约等 20+ 种常见陷阱

2. **租金参考引擎**
   - 输入小区名/地址
   - 展示片区均价对比
   - 输出示例：「该小区均价¥3500，此房源¥3800偏贵8%」
   - 基于链家/贝壳公开数据

3. **看房检查清单**
   - AR 标注检查项（水压、隔音、窗户朝向）
   - 照片记录 + 问题备注
   - 一键生成看房报告

4. **维权话术生成**
   - 场景：押金被扣、房东不退租、维修拖延
   - 基于法律知识库 + 案例学习
   - 一键复制专业交涉文案

### MVP 功能（Phase 1）

- 合同风险扫描（前 3 份免费）
- 风险条款解释
- 基础改写建议

## 技术方案

### 数据层

```
├── 合同风险条款库
│   ├── 违约金条款
│   ├── 维修责任条款
│   ├── 提前解约条款
│   └── 隐形收费条款
├── 租金数据库
│   ├── 小区均价
│   ├── 户型分布
│   └── 历史成交记录
└── 法律知识库
    ├── 租赁法条文
    ├── 典型案例
    └── 维权话术模板
```

### 逻辑层

```
合同扫描服务
├── OCR 文字识别（kimi-k2.5 / GPT-4V）
├── 条款分割与分类
├── 风险识别引擎（规则 + LLM）
└── 改写建议生成

租金参考服务
├── 地址解析（地图 API）
├── 数据爬取/清洗
├── 相似房源匹配
└── 均价计算与对比

维权话术服务
├── 场景识别
├── 法律知识检索
├── 话术生成
└── 语气调整
```

### 展示层

```
前端架构
├── 移动端优先（微信小程序 / H5）
├── 相机集成（拍照扫描）
├── 结果展示（卡片式布局）
└── 一键复制/分享

交互设计
├── 扫描页：相机取景框 + 拍照按钮
├── 结果页：风险高亮 + 解释 + 建议
├── 对比页：租金柱状图 + 百分比
└── 话术页：场景选择 + 生成按钮
```

## 实现步骤

### Phase 1: MVP（4 周）

1. **Week 1-2: 合同扫描核心**
   - 搭建 OCR + LLM 流程
   - 构建风险条款库（10 种常见陷阱）
   - 实现风险识别 + 解释生成

2. **Week 3: 前端开发**
   - 微信小程序基础框架
   - 相机拍照功能
   - 结果展示页面

3. **Week 4: 测试与优化**
   - 真实合同测试（50 份）
   - 识别准确率优化（目标 > 85%）
   - 性能优化（扫描 < 5 秒）

### Phase 2: 租金参考（2 周）

1. 数据爬取与清洗
2. 地址解析服务集成
3. 均价对比算法

### Phase 3: 维权话术（2 周）

1. 法律知识库构建
2. 场景识别与话术生成
3. 用户体验优化

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 租金数据源不稳定（链家/贝壳不开放 API） | 1. 使用公开数据 + 定期爬取<br>2. 用户众包数据补充<br>3. 逐步建立自有数据库 |
| 维权话术需求频次低，无法形成高频习惯 | 1. MVP 聚焦合同扫描（高频刚需）<br>2. 维权作为增值服务<br>3. 增加看房清单等日常功能 |
| AR 看房清单实现成本高 | 1. Phase 1 仅做基础清单（非 AR）<br>2. AR 功能作为 Phase 2 探索 |
| 法律风险（AI 话术可能不准确） | 1. 明确免责声明<br>2. 人工审核高频话术<br>3. 建立用户反馈机制 |
| 竞品模仿 | 1. 快速迭代建立壁垒<br>2. 积累用户数据优化算法<br>3. 建立品牌认知 |
