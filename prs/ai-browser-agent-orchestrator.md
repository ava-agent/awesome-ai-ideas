# AI 浏览器代理调度器 — 智能网页任务编排系统

> **一句话卖点**：运营人员每天花 2 小时在多个网站间切换做重复操作，用调度器定义一次任务 DSL，10 个 Agent 并行执行，从 2 小时变成 5 分钟

## 概述

基于 Issue #153 创建。解决多 AI 浏览器 Agent 协调痛点——现有方案（browser-use、Playwright）是单 Agent，多任务场景需要手动串联。本系统提供：

- **任务 DSL**：用 YAML/JSON 定义多步骤网页任务
- **智能调度器**：根据任务类型自动分配给最合适的 Agent
- **资源隔离**：每个 Agent 独立 browser context，避免状态冲突
- **优先级队列**：紧急任务插队，不阻塞正常流程

## 功能设计

### 核心功能

1. **任务定义层**
   - 可视化任务编辑器（拖拽式）
   - YAML/JSON DSL 语法
   - 支持条件分支、循环、变量传递
   - 任务模板市场（社区分享）

2. **智能调度层**
   - 任务类型识别（数据采集 vs 自动化操作 vs 监控）
   - Agent 能力匹配（哪个 Agent 擅长处理这类任务？）
   - 负载均衡（避免某个 Agent 过载）
   - 失败重试 + 备选 Agent 切换

3. **执行监控层**
   - 实时任务状态仪表盘
   - 异常检测（页面结构变化、反爬触发）
   - 断点续传（长任务失败后从断点恢复）
   - 执行日志 + 回放

4. **协作层**
   - Agent 间上下文共享（MCP 协议）
   - 结果聚合（多个 Agent 的输出合并）
   - 冲突检测（两个 Agent 同时操作同一网站）

## 技术方案

### 数据层

- **任务存储**：PostgreSQL + JSONB（任务 DSL）
- **执行状态**：Redis（实时状态）+ TimescaleDB（历史记录）
- **会话管理**：Redis（browser context 持久化）
- **任务队列**：Redis Streams（优先级队列）

### 逻辑层

```
任务提交 → DSL 解析 → 任务分析 → Agent 匹配 → 资源分配 → 执行监控 → 结果聚合
```

**核心组件**：

1. **DSL Parser**：YAML/JSON → 内部任务图
2. **Task Analyzer**：识别任务类型、估算资源需求
3. **Agent Matcher**：基于历史表现匹配最佳 Agent
4. **Resource Manager**：管理浏览器实例池
5. **Execution Engine**：Playwright + browser-use 集成
6. **Monitor**：实时监控 + 异常检测

### 展示层

- **Web UI**：React + TypeScript
- **任务编辑器**：Monaco Editor（代码模式）+ React Flow（可视化模式）
- **监控仪表盘**：Grafana 或自研
- **CLI**：支持脚本化调用

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 反爬对抗 | Agent 被封禁 | 代理池 + 请求频率限制 + User-Agent 轮换 |
| 网页结构变化 | 任务失效 | AI 视觉备选（GPT-4V 定位元素）+ 社区维护选择器库 |
| 浏览器实例成本高 | 利润率低 | 智能回收机制 + 按需启动 + 容器化隔离 |
| Agent 间竞态条件 | 数据冲突 | 分布式锁 + 乐观并发控制 + 冲突检测 |
| 任务 DSL 学习曲线 | 用户流失 | 可视化编辑器 + 自然语言生成 DSL |
