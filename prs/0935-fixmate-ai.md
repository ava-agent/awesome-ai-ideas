## 1. 项目概述

### 1.1 背景与痛点

独居青年、新房主、租房族在日常生活中频繁遭遇家电故障——空调不制冷、洗衣机漏水、马桶堵塞、燃气灶打不着火……面对这些问题，大多数人的反应是：

- **不知所措**：不知道问题出在哪，不敢自己动手
- **信息过载**：搜索后信息杂乱，无法判断可靠性
- **维修焦虑**：担心被维修师傅坑，不知道合理价格
- **决策困难**：到底是修还是换？修了能撑多久？
- **保修遗忘**：买了家电忘了保修期，白白浪费免费维修

### 1.2 解决方案

FixMate AI 是一款基于多模态大模型的智能家电故障诊断与维修决策助手。用户只需拍照或拍视频描述问题，AI即可：

1. **智能诊断**：识别故障原因，给出置信度
2. **决策建议**：修还是换？给出成本效益分析
3. **DIY指南**：简单问题提供分步维修教程
4. **费用预估**：基于本地化数据库给出合理价格区间
5. **师傅匹配**：推荐附近靠谱的维修服务商
6. **资产管理**：追踪家电保修状态，到期自动提醒

### 1.3 核心价值主张

| 维度 | 传统方式 | FixMate AI |
|------|---------|------------|
| 诊断 | 百度搜索、打电话问 | 拍照秒出结果 |
| 决策 | 凭感觉、问朋友 | 数据驱动的修/换分析 |
| 费用 | 信息不对称、被坑 | 透明价格区间参考 |
| 维修 | 58同城随机找 | 智能匹配、评价透明 |
| 预防 | 坏了才处理 | 预测性维护提醒 |

### 2.2 用户场景

**场景一：凌晨空调罢工**
> 小李凌晨2点发现空调不制冷，打开FixMate拍了空调面板和遥控器的照片。AI诊断为"缺氟+滤网堵塞"，给出两个方案：自己清洗滤网（免费，5分钟教程）+预约加氟（参考价150-200元）。小李先按教程清洗了滤网，温度明显下降，第二天再预约加氟。

**场景二：洗衣机异响**
> 小张听到洗衣机发出奇怪噪音，拍了段视频上传。AI分析后判断是"轴承磨损"，置信度85%。AI给出成本分析：维修约300-500元，新机约1500-2500元。考虑到机器已使用7年，AI建议更换，并推荐了三款性价比机型。

**场景三：租房维修责任**
> 小王租的房子马桶漏水，不确定该谁出钱修。FixMate诊断后标注了故障类型属于"正常老化磨损"，并提示：根据《住房租赁合同》一般条款，此类维修通常由出租方承担。同时生成了维修沟通模板发给房东。

### 3.2 智能诊断流程

```
用户拍照/视频
    ↓
多模态模型初步分析
    ↓
┌─ 高置信度(>80%) → 直接给出诊断结果
│
├─ 中置信度(50-80%) → 追问补充信息
│   ├── 请拍侧面照片
│   ├── 请拍铭牌/型号
│   ├── 请描述异常声音
│   └── 请确认使用年限
│
└─ 低置信度(<50%) → 建议人工服务
    └── 转接专业维修师傅
```

### 3.3 维修决策算法

```
输入：故障类型 + 家电信息 + 使用年限 + 维修费用 + 新机价格
    ↓
计算维修性价比指数 (RVI)：
RVI = (新机价格 - 维修费用) / 新机价格 × 剩余寿命系数 × 能效提升系数
    ↓
RVI > 0.6 → 推荐更换
RVI 0.3-0.6 → 建议维修（附更换备选）
RVI < 0.3 → 推荐维修
    ↓
输出：决策建议 + 成本明细 + 推荐机型（如需更换）
```

### 4.2 AI模型选型

| 能力 | 模型选择 | 理由 |
|------|---------|------|
| 图像识别 | GPT-4o / Gemini Pro Vision | 多模态理解能力强，能识别家电型号和故障特征 |
| 视频分析 | Gemini 1.5 Pro | 长视频理解，适合分析运行异响/异常振动 |
| 对话推理 | GPT-4o / Claude 3.5 | 多轮诊断对话，逻辑推理能力强 |

| 决策分析 | 自研规则+ML混合模型 | 维修决策需要确定性规则和成本计算 |
| 文本生成 | GPT-4o | 生成维修指南、沟通模板等 |

### 4.3 RAG知识库构建

```
知识来源：
├── 品牌官方维修手册 (50+品牌)
├── 用户维修社区内容 (知乎/小红书/B站)
├── 维修师傅经验库 (合作师傅提供)
├── 国家家电标准 (GB系列)
├── 价格数据库 (线上线下采集)
└── 用户反馈与纠错 (持续更新)

知识处理：
原始文档 → 清洗/去重 → 分块 → 向量化 → 向量数据库
                                      ↓
                              元数据标注
                              (品牌/型号/故障类型/难度)
```

### 4.4 数据库设计（核心表）

```sql
-- 家电资产表
CREATE TABLE appliances (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL,
    brand VARCHAR(100),
    model VARCHAR(100),
    category VARCHAR(50),
    purchase_date DATE,
    warranty_end DATE,
    status VARCHAR(20),
    created_at TIMESTAMP
);

-- 诊断记录表
CREATE TABLE diagnoses (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL,
    appliance_id UUID,
    symptoms TEXT,
    image_urls JSONB,
    diagnosis_result JSONB,
    confidence FLOAT,
    recommendation TEXT,
    created_at TIMESTAMP
);

-- 价格数据库
CREATE TABLE price_references (
    id UUID PRIMARY KEY,
    appliance_category VARCHAR(50),
    fault_type VARCHAR(100),
    city_tier INT,
    part_cost_range JSONB,
    labor_cost_range JSONB,
    updated_at TIMESTAMP
);
```
