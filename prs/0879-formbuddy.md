# FormBuddy - AI驱动的政务生活事务一键处理助手

## 项目概述

FormBuddy 是一款专为独居年轻人打造的 AI 驱动政务生活事务处理助手。通过智能识别、自动填表、路线规划等功能，帮助用户高效办理各类政务和生活事务，告别繁琐流程和重复跑腿。

## 痛点分析

### 目标用户画像
- **独居年轻人**（22-35岁），刚步入社会或独自在大城市生活
- 对政务流程不熟悉，缺乏经验指导
- 工作日时间紧张，周末不愿浪费在排队办事上
- 经常因为材料不齐、填错表格而白跑一趟

### 核心痛点
1. **不知道办什么事**：户口迁移、社保转移、居住证办理……种类繁多，不知从何入手
2. **不知道去哪办、怎么办**：各部门网址分散，流程说明晦涩难懂
3. **材料总是准备不齐**：跑三四趟才能办完一件事
4. **表格不会填**：政务表格术语多，填错被退回
5. **排队浪费时间**：到现场才发现排号已满，白跑一趟

## 核心功能

### 1. 智能事项识别

用户只需拍照或语音描述需求，AI 自动匹配对应政务事项。

**工作流程：**
```
用户输入（拍照/语音/文字）
    ↓
意图理解引擎（GLM/GPT-4）
    ↓
政务事项知识库匹配
    ↓
返回：事项名称 + 办理渠道 + 所需材料清单 + 预计时间
```

**示例交互：**
- 用户拍照身份证 + 租房合同 → 识别：居住证新办
- 语音："我刚换了工作，想把社保转过来" → 识别：社保转移接续
- 文字："我要开个证明" → 引导追问：什么证明？用途是什么？

**技术实现：**
- 多模态输入解析（图片OCR + 语音ASR + 文本NLU）
- 意图分类模型：基于 GLM-4 微调的政务意图分类器
- 知识图谱：构建全国政务事项知识图谱，支持模糊匹配和智能推荐

### 2. 材料清单生成器

根据用户所在城市和具体事项，自动生成个性化材料准备清单。

**特色功能：**
- **城市适配**：不同城市材料要求不同，自动匹配用户所在城市的最新政策
- **已有材料智能识别**：扫描用户已有证件，自动标记已备/未备材料
- **替代方案推荐**：缺少某项材料时，推荐替代证明方式
- **有效期提醒**：证件过期自动提醒续期

**材料清单示例（居住证办理-北京）：**
| 序号 | 材料名称 | 状态 | 备注 |
|------|---------|------|------|
| 1 | 身份证原件及复印件 | 已有 | - |
| 2 | 近期免冠照片 | 未备 | 建议提前拍摄 |
| 3 | 房屋租赁合同 | 已有 | 需在有效期内 |
| 4 | 房东房产证复印件 | 未备 | 需联系房东 |
| 5 | 社保证明 | 需更新 | 可在线打印 |

### 3. AI 填表助手

通过 OCR 和 NLP 技术，自动识别表格字段并辅助填写。

**工作流程：**
```
获取政务表格（拍照/下载PDF）
    ↓
OCR 表格结构识别
    ↓
字段语义理解 + 用户信息自动匹配
    ↓
AI 预填写（高亮待确认项）
    ↓
用户确认/修改
    ↓
生成可打印/提交的完整表格
```

**核心技术：**
- 表格 OCR：基于 PaddleOCR 的复杂表格识别，支持手写体
- 字段映射：建立用户信息库与政务表格字段的映射关系
- 智能校验：填写完成后自动校验格式、逻辑一致性
- 电子签章：支持电子签名和盖章（合规前提下）

### 4. 办事路线规划

将多个待办事项整合，规划最优办理路线。

**功能亮点：**
- **智能排序**：根据办事地点距离、营业时间、排队预估，生成最优办理顺序
- **实时排队信息**：接入各政务大厅排队叫号系统，显示实时等待人数
- **预约提醒**：自动预约热门事项，避免现场长时间等待
- **路线导航**：支持步行/公交/驾车多种出行方式
- **时间预估**：精确预估每个事项的办理时长和路程时间

**路线规划示例：**
```
上午 9:00 - 9:30  | 出发 → 街道办事处
上午 9:30 - 10:00 | 办理居住证（当前排队 12 人，约 25 分钟）
上午 10:00 - 10:15 | 前往社保局
上午 10:15 - 11:00 | 办理社保转移（已预约 10:30）
上午 11:00 - 11:30 | 附近打印店补材料
```

### 5. 进度追踪与提醒

全程跟踪办事进度，关键节点主动推送提醒。

**追踪维度：**
- **材料准备进度**：待备/准备中/已完成
- **办理进度**：未开始/进行中/已完成/需补充
- **证件有效期**：到期前 30/15/7 天分级提醒
- **政策变更**：关注事项的政策更新通知

**提醒方式：**
- 微信服务号推送
- 小程序内通知
- 短信提醒（重要事项）
- 日历同步（支持系统日历）

### 核心技术选型

| 模块 | 技术方案 | 说明 |
|------|---------|------|
| 意图理解 | GLM-4 / GPT-4 API | 多轮对话，精准理解用户需求 |
| 知识库 | RAG（检索增强生成） | 城市政务知识库，向量检索 |
| 向量数据库 | Milvus / Qdrant | 高性能向量相似度检索 |
| OCR引擎 | PaddleOCR + 自研模型 | 政务表格专用 OCR 优化 |
| 后端框架 | Python FastAPI | 高性能异步 API 服务 |
| 小程序 | Taro + React | 跨平台小程序开发 |
| 缓存 | Redis | 排队信息缓存、会话管理 |
| 数据库 | PostgreSQL | 用户数据、办事记录存储 |
| 搜索 | Elasticsearch | 政务事项全文检索 |
| 消息队列 | RabbitMQ | 异步任务处理（OCR、生成等） |
| 对象存储 | MinIO | 证件图片、表格文件存储 |

### 政务知识库建设

**数据来源：**
- 各省市政务服务网公开信息
- 国务院客户端办事指南
- 地方政务服务APP公开数据
- 用户贡献的办事经验

**知识库结构：**
```
政务事项
├── 事项基本信息（名称、编码、分类）
├── 办理条件（适用人群、前置条件）
├── 办理材料（材料清单、模板、样例）
├── 办理流程（步骤说明、时限）
├── 办理地点（线上渠道、线下窗口信息）
├── 收费标准
├── 常见问题 FAQ
└── 政策法规依据
```

**更新机制：**
- 定期爬取政务服务网更新（每周）
- 用户反馈纠错机制
- 政策变更自动检测与推送

## 风险与应对

### 合规风险
- **风险**：政务数据敏感性，涉及个人信息保护
- **应对**：严格遵循《个人信息保护法》，数据本地化存储，最小化采集原则

### 数据准确性
- **风险**：政务政策频繁变更，知识库可能过时
- **应对**：建立多源校验机制，用户反馈纠错奖励，定期人工审核

### 竞争压力
- **风险**：支付宝/微信可能加强政务功能
- **应对**：深耕 AI 能力差异化，建立用户办事经验社区壁垒

### 技术风险
- **风险**：OCR 在复杂表格上的准确率
- **应对**：针对政务场景专项训练，人机协作兜底
