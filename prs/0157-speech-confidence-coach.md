# AI 演讲自信教练

> **一句话卖点**：职场新人在重要汇报前 30 分钟，用 AI 演讲教练陪练开场白，从上台前腿抖到从容控场全场倾听

## 概述

基于 Issue #157 创建。演讲恐惧是仅次于死亡恐惧的第二大恐惧源（75% 的人害怕公开演讲）。本产品通过 AI 陪练、卡壳点预测、模拟提问、语音反馈等功能，帮助用户从「上台前腿抖」到「从容控场」。

**核心价值**：不是教你演讲理论，而是陪你在实战中练习，给具体、即时、客观的反馈。

## 功能设计

### 核心功能
1. **开场白特训** - 开场前 30 秒最紧张，AI 陪练直到能脱稿流利说出
2. **卡壳点识别** - 分析演讲稿，预测容易忘词的地方，提供记忆锚点
3. **语音反馈** - 检测语速、停顿、语气词（「然后」「那个」），给出改进建议
4. **模拟提问训练** - 根据内容生成可能的 Q&A，练习临场反应
5. **紧张管理** - 提供呼吸练习、正念技巧，缓解上台前焦虑

### 差异化优势
| 现有方案 | 本产品 |
|---------|--------|
| 演讲培训班 ¥5000+，需要周末 2 天 | 随时随地练习，每次 10-30 分钟 |
| 找同事练，怕丢脸，反馈不专业 | AI 客观分析，零尴尬，专业反馈 |
| 看视频学习理论，缺少实战 | 实战模拟，真实提问，即时反馈 |
| 录音自己听，发现不了问题 | AI 检测语速、语气词、停顿，精准定位 |
| 演讲前去厕所深呼吸 | 提供科学的紧张管理技巧 |

## 技术方案

### 数据层
- **演讲稿存储**：用户上传的 PPT/演讲稿（加密存储）
- **练习记录**：每次练习的语音数据、分析结果（本地优先）
- **用户画像**：卡壳模式、语气词偏好、进步曲线

### 逻辑层
```
演讲稿输入 → NLP 分析 → 识别卡壳点 → 生成记忆锚点
     ↓
语音输入 → Whisper 识别 → 语速/停顿/语气词检测 → 生成反馈报告
     ↓
内容理解 → LLM 生成 → 模拟提问 → Q&A 训练
```

### 展示层
- **移动端**：随时随地练习（核心场景）
- **Web 端**：详细分析报告、历史记录
- **耳机模式**：实时语音反馈

### 技术栈
- 语音识别：Whisper API / 讯飞语音
- 文本分析：GLM-4 / kimi-k2.5
- 语音合成：Azure TTS / 讯飞 TTS
- 前端：React Native（跨平台）
- 后端：Node.js + PostgreSQL

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 使用场景低频，用户「学会后流失」 | 延伸到「日常会议发言」「面试」等高频场景 |
| 效果难量化，用户不知道「进步了」 | 设计可量化指标（语速稳定性、语气词减少量） |
| 语音识别准确率、实时性要求高 | Phase 1 先做「上传录音分析」，降低技术门槛 |
| 通用 AI 也能练演讲，差异化不足 | 强调「开场白特训」「卡壳点预测」等独有功能 |
| 用户可能被「实时陪练」绑架 | 提供「渐进式放手」机制，培养独立能力 |

## 目标用户

- **核心用户**：职场新人、求职者、需要公开演讲的专业人士
- **扩展用户**：学生（答辩）、销售（话术）、教师（授课）
