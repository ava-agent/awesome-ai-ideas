# 🍃 WildBite — AI 驱动的城市野生食物发现与安全觅食指南

> **Issue**: #878  
> **目标用户**: 户外爱好者、自然探索者、亲子家庭  
> **核心价值**: 让城市居民安全地发现、识别、采集和烹饪身边的野生食物

---

## 一、概述

### 1.1 项目愿景

WildBite 致力于成为**最安全的 AI 觅食伴侣**，帮助城市居民重新发现身边的自然馈赠。通过多模态 AI 识别、季节性地图和社区协作，让采集野生食物变得既安全又有趣。

### 1.2 核心问题

| 痛点 | 描述 | 严重程度 |
|------|------|----------|
| 误食中毒 | 每年数千人因误采有毒植物就医 | 🔴 致命 |
| 知识断层 | 城市化导致传统采食知识失传 | 🟠 高 |
| 识别困难 | 有毒品种与可食用品种外观极度相似 | 🔴 致命 |
| 信任缺失 | 现有工具准确率不足，用户不敢依赖 | 🟠 高 |
| 体验断裂 | 从发现到烹饪缺乏完整链路 | 🟡 中 |

### 1.3 产品定位

```
安全觅食 = AI识别 + 专家知识库 + 社区验证 + 应急保障
```

WildBite 不是"拍照识花"类应用的简单延伸，而是一个**以安全为第一原则**的完整觅食解决方案。

#### 3.2.1 多模态安全识别系统

```python
class SafetyIdentifier:
    """三级识别 + 安全验证引擎"""
    
    def identify(self, image, location, season):
        # 第一级：多模态 LLM 快速识别
        candidates = self.multimodal_llm.identify(image)
        
        # 第二级：专业植物模型精排
        refined = self.plant_model.rank(candidates, image)
        
        # 第三级：安全规则引擎校验
        for plant in refined:
            safety = self.safety_engine.check(
                plant=plant,
                location=location,
                season=season,
                lookalikes=self.db.get_toxic_lookalikes(plant)
            )
            plant.safety_rating = safety.rating  # 🟢/🟡/🔴
            plant.toxic_lookalikes = safety.lookalikes
            plant.edibility_conditions = safety.conditions
        
        return refined
```

**识别结果展示规范**:

| 俗名 | 地方俗称 | 蒲公英、婆婆丁 |

```sql
-- 食谱匹配查询
SELECT r.* FROM recipes r
JOIN recipe_ingredients ri ON r.id = ri.recipe_id
JOIN plants p ON ri.plant_id = p.id
WHERE p.id IN (:identified_plants)
  AND p.edible_part IN (r.required_parts)
  AND r.season @> ARRAY[:current_season]
  AND r.difficulty <= :user_skill_level
ORDER BY 
  r.rating DESC,
  r.match_score DESC
LIMIT 10;
```

#### 3.2.5 安全第一机制

| 安全层级 | 机制 | 实现方式 |
|----------|------|----------|
| **L1 预防** | 毒性分级展示 | AI 识别结果强制展示安全等级 |
| **L2 警告** | 相似种对比 | 有毒相似种高亮提醒，强制确认 |
| **L3 拦截** | 高危物种阻断 | 已知剧毒物种采集前弹出警告 |
| **L4 应急** | 一键求助 | 直连全国中毒控制中心热线 |
| **L5 保障** | 保险对接 | 采集意外保险（可选购买） |

```python
class SafetyEngine:
    """安全规则引擎"""
    
    # 毒性等级定义
    TOXICITY_LEVELS = {
        'EDIBLE': {
            'color': '🟢',
            'label': '可食用',
            'action': 'none'
        },
        'CONDITIONAL': {
            'color': '🟡', 
            'label': '有条件可食',
            'action': 'show_conditions',
            'conditions': ['烹饪处理', '限量食用', '特定部位']
        },
        'CAUTION': {
            'color': '🟠',
            'label': '谨慎处理',
            'action': 'show_warnings',
            'require_confirmation': True
        },
        'TOXIC': {
            'color': '🔴',
            'label': '有毒，禁止食用',
            'action': 'block_and_warn',
            'show_lookalikes': True,
            'show_antidote': True
        },
        'DEADLY': {
            'color': '💀',
            'label': '剧毒，可能致命',
            'action': 'block_emergency',
            'auto_contact_poison_control': True
        }
    }
    
    def check(self, plant, location, season, lookalikes):
        # 检查当前季节是否可食
        if not self._is_in_season(plant, season):
            return self._create_warning('非推荐采集季节')
        
        # 检查是否有毒相似种
        toxic_matches = self._find_toxic_lookalikes(plant, location)
        if toxic_matches:
            return self._create_toxic_warning(plant, toxic_matches)
        
        # 检查地点安全性（污染区、保护区等）
        location_check = self._check_location_safety(location)
        if not location_check.safe:
            return self._create_location_warning(location_check)
        
        return self._create_safe_result(plant)
```

## 四、用户冷启动策略

### 4.1 冷启动挑战

新用户面临三个核心问题：
1. **不信任**：对 AI 识别结果持怀疑态度
2. **不知道去哪**：缺乏采集地点信息
3. **不敢尝试**：担心误食中毒

### 4.2 分阶段冷启动方案

```
Phase 1: 信任建立（第1周）
├── 引导用户识别常见安全植物（蒲公英、荠菜、马齿苋）
├── 每次识别展示"置信度"和"验证来源"
├── 完成5次识别 → 解锁"觅食入门"徽章
└── 推送安全小贴士（每日一条）

Phase 2: 地点发现（第2-3周）
├── 根据用户位置推荐附近安全采集点
├── 展示社区已验证的采集报告
├── 提供周末觅食路线规划
└── 邀请参加线下自然教育活动

Phase 3: 社区融入（第4周+）
├── 鼓励分享采集成果
├── 参与物种分布验证
├── 获取"认证采集者"标识
└── 参与社区活动和挑战赛
```

### 4.3 增长飞轮

```
  更多用户采集

  更丰富的地点数据 ←→ 更准确的季节模型
       ↓                    ↑

       ↓
  社区内容沉淀
       ↓
  吸引新用户加入  ──→ 循环
```

### 5.4 数据库设计核心表

```sql
-- 植物物种表
CREATE TABLE plants (
    id UUID PRIMARY KEY,
    scientific_name VARCHAR(200) NOT NULL,
    common_names JSONB,           -- {"zh": ["蒲公英", "婆婆丁"], "en": ["Dandelion"]}
    family VARCHAR(100),
    genus VARCHAR(100),
    toxicity_level VARCHAR(20) NOT NULL,  -- EDIBLE/CONDITIONAL/CAUTION/TOXIC/DEADLY
    edible_parts JSONB,           -- ["leaf", "flower", "root"]
    seasonal_availability JSONB,  -- {"spring": true, "summer": false, ...}
    nutritional_info JSONB,
    habitat_info TEXT,
    distribution JSONB,           -- 省级分布数据
    pharmacopoeia_ref VARCHAR(50), -- 中国药典编号
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 有毒相似种关系表
CREATE TABLE toxic_lookalikes (
    id UUID PRIMARY KEY,
    edible_plant_id UUID REFERENCES plants(id),
    toxic_plant_id UUID REFERENCES plants(id),
    similarity_description TEXT,   -- "叶片形状相似，但泽漆茎部有白色乳汁"
    differentiation_tips JSONB,    -- 区分要点列表
    risk_level VARCHAR(20),
    image_comparison_url TEXT      -- 对比图
);

-- 采集点表
CREATE TABLE foraging_spots (
    id UUID PRIMARY KEY,
    location GEOGRAPHY(POINT, 4326) NOT NULL,
    city VARCHAR(50),
    district VARCHAR(50),
    species_ids UUID[],
    seasonal_status JSONB,
    safety_rating VARCHAR(20),
    pollution_check JSONB,         -- 环境检测数据
    report_count INTEGER DEFAULT 0,
    last_verified_at TIMESTAMP,
    created_by UUID,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 七、总结

WildBite 通过 **AI + 专家知识 + 社区** 的三角验证体系，在保证绝对安全的前提下，为城市居民打开了一扇通往自然美食的大门。MVP 阶段聚焦于**小程序 + 核心识别 + 基础地图**，快速验证用户需求后逐步扩展至全平台、全链路的觅食生态。

**核心理念：敬畏自然，安全为先，科技赋能，人人可享。**
